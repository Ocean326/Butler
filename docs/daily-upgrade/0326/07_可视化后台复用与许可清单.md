## 1. 复用原则

1. 只允许 `选择性 vendoring`，禁止整体 fork。
2. 所有复用必须记录来源与许可。
3. 禁止引入有竞争限制或 SaaS 限制的代码。
4. Flowise 仅允许复用 Apache 许可范围内的前端代码，禁止触碰 enterprise 目录。

## 4. 许可与归档要求

1. 任何 vendored 代码必须附带 `LICENSE` 或 `NOTICE` 摘要。
2. 复用必须在控制台代码目录中维护 `attribution` 列表。
3. 未在清单中的第三方项目，默认禁止直接复制。
