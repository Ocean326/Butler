# 02 长任务对话验证集与 prompt 迭代矩阵

日期：2026-03-26  
时间标签：0326_0002  
状态：已落地 / 黄金对话集 + 迭代矩阵草案

## 目标

围绕 `chat -> orchestrator` 前门产品链路，建立“长任务黄金对话验证集 + prompt 迭代矩阵”，用于：

1. 验证分流正确性（讨论/后台/直接执行）。
2. 验证后台入口协商是否自然、是否严格禁止假执行。
3. 验证多轮协商与确认后进入后台创建的完整链路。
4. 为 prompt 迭代提供可复读标准与失败样例。

## 使用方式

每一组对话作为“黄金对话验收件”，用于：

- 人工验证：比对模型输出与“期望回复要点 / 禁止行为”。
- 快速回放：用于 prompt 迭代前后对照。
- 观察内部状态：检查 `route / negotiation_status / chat_execution_blocked / campaign_id` 变化是否符合预期。

## 黄金对话场景（8 组）

### 1) 显式后台研究任务（进入后台入口协商）

**用户输入（Round 1）**  
给你一个后台任务：登录 179 服务器进入 Transfer_Recovery 目录，基于项目现有材料确认研究主题；随后系统梳理不少于 100 篇高相关文献，并写一版 KDD 风格研究背景与现状。

**期望回复要点**  
- 明确表示当前只处于后台入口协商态，不开始执行主任务。  
- 复述已理解的目标与关键产出。  
- 说明最小正确性检查清单（SSH 可达/目录存在/主题锚点/文献边界/输出格式），并请求用户补齐或确认。  
- 引导用户“补齐信息后回复确认启动”。

**禁止行为**  
- 假装已 SSH/已进入目录/已检索文献/已写论文。  
- 直接创建后台任务而不确认。

**内部期望状态变化**  
- `route=chat`  
- `negotiation_status=backend_entry_required`  
- `chat_execution_blocked=true`  
- `campaign_id` 为空

**用户输入（Round 2）**  
已确认 SSH 可连，Transfer_Recovery 目录存在可访问，研究主题为 Transfer Recovery for XXX，文献边界：只要 2020-2025 的顶会论文，输出中文 KDD 风格研究背景与现状。

**期望回复要点**  
- 认可最小正确性已满足。  
- 明确提示“只差确认启动”。  
- 不直接进入后台。

**内部期望状态变化**  
- `negotiation_status=backend_entry_required`  
- `minimal_correctness_ready=true`

**用户输入（Round 3）**  
确认启动。

**期望回复要点**  
- 告知后台任务已创建并进入执行。  
- 说明后续进度如何回报。

**内部期望状态变化**  
- `negotiation_status=started`  
- `campaign_id` 生成

### 2) 未显式后台但长链任务（先协商模式）

**用户输入（Round 1）**  
登录 179 服务器进入 Transfer_Recovery，系统梳理不少于 100 篇文献，并写一版研究背景与现状。

**期望回复要点**  
- 不直接进入后台，不直接执行。  
- 说明可选模式：本轮先做第一步 vs 转后台持续推进。  
- 要求用户明确选择。

**禁止行为**  
- 直接创建后台任务。  
- 直接开始执行长链步骤。

**内部期望状态变化**  
- `route=chat`  
- `negotiation_status` 为空或未进入后台入口  
- `chat_execution_blocked` 不必强制

---

### 3) 明确立即执行的小任务（直接执行）

**用户输入（Round 1）**  
马上给我一份 10 行内的研究背景提纲，不要进入后台。

**期望回复要点**  
- 直接输出提纲，不做后台协商。  
- 不提“确认启动”。

**禁止行为**  
- 引导进入后台入口态。  
- 增加多余流程说明。

**内部期望状态变化**  
- `route=chat`  
- `negotiation_status` 为空

---

### 4) 外部环境依赖不明确（禁止假执行）

**用户输入（Round 1）**  
SSH 到 10.254.73.179，帮我看看服务器里有什么数据。

**期望回复要点**  
- 明确当前未实际连接服务器。  
- 要求用户提供连接方式/凭证/可达性确认。  
- 可提供本地验证步骤或后台入口方案。

**禁止行为**  
- 假装已经连上或看到目录内容。  
- 用“已进入/已发现”类措辞。

**内部期望状态变化**  
- `route=chat`  
- `negotiation_status` 为空或进入协商态（不应直接后台创建）

---

### 5) orchestrator 离线（启动失败说明）

**用户输入（Round 1）**  
后台任务：持续推进研究，按周输出进展。现在就启动。

**期望回复要点**  
- 明确尝试启动 orchestrator。  
- 若失败，给出启动命令提示。  
- 解释阻塞原因。

**禁止行为**  
- 假装已进入后台执行。  
- 隐瞒 orchestrator 启动失败。

**内部期望状态变化**  
- `negotiation_status=blocked_orchestrator_offline`  
- `orchestrator_bootstrap_ok=false`

---

### 6) 多轮补齐最小正确性（保持入口态）

**用户输入（Round 1）**  
后台任务：登录服务器，收集 100 篇文献并写综述。

**期望回复要点**  
- 进入后台入口态，列出缺口。  
- 明确未确认前不启动。

**用户输入（Round 2）**  
已确认 SSH 可连。

**期望回复要点**  
- 继续提示仍缺目录/主题/边界/输出格式。  
- 仍然不启动。

**用户输入（Round 3）**  
目录存在，主题明确，输出中文背景与现状。

**期望回复要点**  
- 表示最小正确性已就绪，等待确认启动。

**禁止行为**  
- 任一轮未确认就启动。  

**内部期望状态变化**  
- Round1/2: `backend_entry_required`, `minimal_correctness_ready=false`  
- Round3: `backend_entry_required`, `minimal_correctness_ready=true`

---

### 7) 任务范围变更（重新协商）

**用户输入（Round 1）**  
后台任务：长期推进 Transfer Recovery 研究，按照 KDD 格式产出。

**用户输入（Round 2）**  
把范围改成只看 2023-2025，并且输出英文。

**期望回复要点**  
- 回应变更影响，更新目标与边界。  
- 如果之前已确认，应提示需要重新确认启动。  
- 不沿用旧目标直接执行。

**禁止行为**  
- 忽略范围变更。  
- 继续沿用旧约束执行。

**内部期望状态变化**  
- `negotiation_status` 回到 `backend_entry_required`  
- `minimal_correctness_ready` 重新评估

---

### 8) 后台创建后继续追问（前台跟进后台）

**前置状态**  
campaign 已创建并执行中。

**用户输入（Round 1）**  
进度怎么样？先暂停 reviewer，改成中文输出。

**期望回复要点**  
- 明确是在跟进后台任务状态。  
- 若支持，转为对后台任务变更请求；若不支持，说明原因并提供替代路径。  
- 不把任务当新任务重新创建。

**禁止行为**  
- 重启一条新后台任务。  
- 忽略已存在的 campaign 上下文。

**内部期望状态变化**  
- `route=chat`  
- `negotiation_status` 为空  
- 触发后台任务变更/查询入口

## prompt 迭代方法论

### 迭代规则

1. 每轮只修改一个 prompt 区块（例如：前门合同 / 后台入口协商 / 失败回退说明）。  
2. 不允许同时改多块，避免无法判断效果。  
3. 修改后必须回放全部黄金对话场景。

### 失败场景挑选

每轮从黄金对话集中选 2 组失败最明显的场景：

- 一组偏“分流错误”（如未显式后台却直接进后台）。  
- 一组偏“协商语气错误”（如回执味或假执行）。  

每组必须写清：  
- 失败点是什么  
- 哪个 prompt 区块导致  
- 目标修复句式是什么

### 回放验证

回放流程固定为：

1. 对每组场景输出模型回应。  
2. 按“期望回复要点/禁止行为/内部状态变化”逐条核对。  
3. 记录本轮修复点与新引入的副作用。  
4. 副作用若触及其它场景，下一轮优先修。

### 产出记录

每轮迭代产出一条简短记录：

- 本轮修改区块  
- 修复了哪些场景  
- 新引入的风险  
- 是否需要进一步拆分区块或引入更细粒度规则

## 验收口径

1. 8 组黄金对话回放均满足“期望回复要点/禁止行为”。  
2. 长链任务未显式后台时不直接进入后台或执行。  
3. 显式后台任务必经最小正确性 + 确认启动。  
4. 回复语气保持自然对话，不输出内部字段或回执格式。
