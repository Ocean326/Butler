## 当前完整系统运行图

### 1. 四层职责

1. 第 `4` 层 `domain/product/frontdoor plane`
   - `chat/mainline.py`
   - `chat/negotiation.py`
   - `chat/task_query.py`
   - 负责：对话入口、协商、查询、后台入口、渠道回显
2. 第 `3` 层 `orchestrator/control plane`
   - `mission_orchestrator / campaign_service / query_service / runner`
   - 负责：mission/node/branch 调度、后台常驻、写回投影、观测窗口
3. 第 `2` 层 `runtime_os/process runtime`
   - `workflow_session / governance / writeback / checkpoint / artifact / blackboard`
   - 负责：跨轮过程真语义与统一 runtime verdict / writeback contract
4. 第 `1` 层 `runtime_os/agent runtime`
   - `cli/native provider adapter`
   - 负责：单次执行适配、provider-native 能力消费、execution fact

### 2. 五个 harness 面

1. `FrontDoor Harness`
   - 对话协商、任务查询、后台入口判定、前台阻断假执行
2. `Control Harness`
   - mission / campaign façade、submit/query/control/feedback
3. `Execution Harness`
   - runner、workflow VM、execution bridge、自动 dispatch/execute、重启恢复
4. `Process Harness`
   - workflow session、governance bridge、统一 writeback、共享状态与产物
5. `Feedback Harness`
   - query/observe、Feishu task doc、push 更新、started-task follow-up

### 3. 一条真实长任务路径

`用户消息 -> RequestIntake -> negotiation/query -> 必要时 bootstrap orchestrator -> create campaign/mission -> runner dispatch/execute -> ProcessExecutionOutcome / ProcessWritebackProjection -> query/observe/feedback_notifier -> chat / Feishu doc / push`

这条路径当前已经不再只是文档设计，而是有代码、测试和 acceptance 证据共同支撑。

## 今天新增的稳定化修复

### 1. 前门与后台反馈面继续收口

1. `chat -> orchestrator` 前门硬闸门继续生效：
   - `chat_execution_blocked=true` 会真实阻断前台误执行
   - `should_discuss_mode_first=true` 会抬升为后台入口触发信号
2. 后台任务启动后，同会话普通补充消息默认改走 `append_feedback`，不再回退成 chat 前台重新执行主任务。
3. 显式 `feedback_contract.doc_enabled=false` 现在会被稳定保留，允许 Feishu 走“只 push、不建 doc”的反馈面，不再在规范化过程中丢失。

### 2. Feishu feedback harness 补齐 push-only 路径

1. `feedback_notifier` 现在在 push-only 模式下仍会：
   - 发送 startup push
   - 发送高信号事件 push
   - 正确把 `push_count` 记入 runner 周期摘要
2. 之前 `doc_enabled=false` 时，push-only 路径会被弱化成“既不建文档，也不稳定上报”的半失效状态；现在这条缺口已补齐。

### 3. runner 启动噪音与示例配置口径统一

1. `orchestrator.interfaces` 改成 lazy export，`python -m butler_main.orchestrator.interfaces.runner` 不再触发包初始化导致的 runpy 噪音警告。
2. 两份示例配置现在和真实稳定运行口径统一：
   - `auto_dispatch=true`
   - `auto_execute=true`
3. 这避免了“代码默认已恢复自动推进，但示例配置仍把 orchestrator 带回空转态”的再次倒退。
