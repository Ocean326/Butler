# Daily Upgrade

`daily-upgrade/` 用于按日期归档阶段性改动文档。

**索引页：[INDEX.md](./INDEX.md)**（由 `gen_index.ps1` 自动生成；新增日期目录或文档后，在仓库根目录执行 `.\docs\daily-upgrade\gen_index.ps1` 即可刷新。）

## 约定

1. 每次新增日更文档先创建 `docs/daily-upgrade/<MMDD>/`
2. 同一天的现状、计划、排查、总结放在同一日期目录
3. 文档内优先使用相对路径，不写本机绝对路径
4. 长期有效内容不要放这里，应迁入 `docs/concepts/`

## 规则

1. 从 2026-03-15 19:00 起，Butler 的变更说明文档默认写具体更新时间，至少精确到小时，推荐写到 `yyyy-MM-dd HH:mm:ss`。
2. 当日持续变更，优先追加到当日现状文档，并按时间块记录，而不是只改“最终结论”不留时间。
3. 后续代码改动若影响主链路、架构边界、测试结果、运行方式或维护规范，必须同步更新对应文档时间戳。
