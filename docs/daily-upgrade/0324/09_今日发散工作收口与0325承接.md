# 09 今日发散工作收口与 0325 承接

日期：2026-03-24
状态：已收口

## 为什么要补这份

今天除了 `Workflow IR / VM / Orchestrator` 主线之外，还做了大量 chat、skill、工作区治理、BrainStorm 侧的发散工作。

如果只看任务面，会显得今天被打散了；但如果按 Butler 的系统层次看，这些发散其实已经压成了 3 类稳定增量。

## 一、内核主线没有偏

`01-04` 依然是今天最重要的骨架：

1. 定义 Butler 的统一 `Workflow IR`
2. 补 `Workflow VM` 的最小执行语义
3. 把 `verification / approval / recovery` 从字段推进到执行语义
4. 让 `orchestrator` 开始从 branch 搬运器转向 workflow control plane

也就是说，今天不是“边聊边散”，而是在先立内部语言，再往外扩执行边界。

## 二、后半程发散实际压出了 4 条可复用侧线

### 1. skill 生命周期第一次闭环

从 `05` 和 `06` 看，今天 skill 体系已经从“搜候选、导仓库”推进到：

1. `candidate shortlist`
2. `incubating intake`
3. `promotion to active`
4. `registry / collection exposure`
5. 命中未转换能力时，优先 promotion，再复用

这条线的价值不只是多了几个 skill，而是 Butler 终于开始具备“把外部能力沉淀成内部可复用资产”的机制。

### 2. 工作区治理从文档约定变成代码真约束

`07` 的核心结论很硬：

1. 这轮工作区失控不是没人写守则，而是守则没有固化成默认路径策略。
2. skill runtime、skill governance、测试 tmp 之前都各自随手落盘，导致 repo 根和 `工作区/` 根层持续长脏源。
3. 本轮已经补出统一路径解析真源，并把：
   - 对外运行结果
   - skill 治理过程态
   - pytest tmp

   做了分层。

这意味着“允许写哪里”开始进入系统约束，而不是继续靠记忆。

### 3. chat 前台链路已经能按统一契约分析

`08` 把一个常见误判讲清楚了：

1. 不是 Butler 没有统一 chat 设计。
2. 而是统一语义层已经有了，但渠道执行层收口程度不同。

现在可以明确概括为：

1. CLI：统一 runtime 的终端文本壳
2. 飞书：当前唯一完整贯通的生产级前台渠道
3. 微信：输入适配和 delivery plan 已有，但真实 transport 还没接完

这条结论很重要，因为它决定后面补的是“能力真源与 execute”，不是重新发明前门。

### 4. `butler_bot_agent/agents/` 还不能被误删

今天对 `agents/` 引用的彻查，解决了一个高风险认知偏差：

1. chat 外壳虽然迁出了一部分
2. 但脑子侧的记忆、子 agent、team、public capability、部分 docs 真源还在 `butler_bot_agent/agents/`

所以当前真实状态更接近：

1. `chat/assets/*` 是前台产品层真源
2. `butler_bot_agent/agents/*` 仍是知识/记忆/协作能力层真源

这会直接影响后续目录迁移和 refactor 的策略，不应该再按“纯历史目录”处理。

## 三、BrainStorm 侧已经给出 0325 的有效输入

今天 research / autoresearch 那组整理，已经不是单纯灵感，而是明天可以直接执行的输入。

压缩后只剩 4 句最重要：

1. `idea 收敛先于编码`
2. `harness 工程先于 agent 炫技`
3. `skill 稳定性` 要按实验对象看，不只看能不能跑
4. `context rot` 是 research agent 的一等故障

对应的明天最小闭环也已经很清楚：

1. 输入题目和资料
2. 产出 proposal
3. reviewer 给可执行修改意见
4. refine 出第二版
5. 全部落盘

## 四、今天真正形成的统一判断

今天看起来分叉很多，但实际形成的是一个更完整的 Butler 图景：

1. 内核层：`Workflow IR -> VM -> governance -> orchestrator`
2. 能力层：`candidate -> incubating -> active skill -> collection`
3. 治理层：`workspace path policy / tmp guard / pollution stop-loss`
4. 前台层：`channel-aware runtime + delivery gap map`
5. research 层：`proposal/review/refine` 先于直接编码

所以 `0324` 不是简单“聊散了”，而是把 Butler 从“内部语言工程”往“能力供给、执行治理、渠道交付、research workflow”四个外延方向各补了一截。
