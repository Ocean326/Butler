# chat 系统性升级文档包

## 目标

本目录用于承接本轮 `butler_main/chat` 的系统性升级沉淀，覆盖：

- 规划
- 代码升级
- 功能升级
- 架构升级
- 具体实施计划

本轮升级的核心方向不是“再补一个渠道适配”，而是把 chat 从“可用的多入口实现”升级为“可持续扩展的统一核心 + 渠道策略 + 常驻服务”体系。

## 本轮升级关键词

- 统一 chat core
- 飞书 / 微信 / CLI 对话解耦
- 常驻 core + 本地 `butler` 客户端
- 渠道 prompt / presentation / delivery policy 下沉
- 多渠道扩展准备
- 多用户会话与长期演进准备

## 当前判断

当前 chat 已经具备以下基础：

- 同一套 `run_agent`、recent memory、long-term memory、skills、runtime
- 飞书 / 微信 / CLI 三个入口可工作
- 微信对话提示词已从飞书提示词中分离
- 本地 `butler-core` / `butler` 常驻核心能力已落下第一版
- 微信多用户 `Phase 1A + 1B + 1D` 已落地：显式会话键、会话注册表、按会话串行分发、core 状态可见

但当前结构仍存在以下问题：

- 渠道知识仍集中在 `channel_profiles.py`
- `mainline.py` 仍直接依赖飞书 / 微信 adapter
- `core.py` 目前是 supervisor，不是纯业务 core
- 新增渠道仍需回改公共层
- 多用户共享对话、多种入口策略、多租户形态尚未结构化

因此，本轮升级目标是把“已经能跑”进一步升级成“后续可持续扩展”。
