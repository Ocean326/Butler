## 一句话裁决

Butler Desktop 当前最准确的现役定义是：

> **Desktop 是一个 `Manager-facing conversation shell over the canonical team runtime`。**
>
> 它不是真正的 `Team OS frontend`，也不是多页 dashboard/workbench；它只把 `TaskContract + Receipt Spine + Recovery Cursor + Authority/Policy` 这条 truth chain，以单个 `Manager thread` 的连续对话形式投影出来。

## 今日新增结论

1. `team` 当前指的是 `canonical team runtime over vendor substrate` 这条运行时主线，不是组织内核，也不是可写 `team graph`。
2. `desktop` 当前只是一层产品表面：它消费 shared surface 和 bridge，不拥有自己的 mission truth、team truth 或 raw sidecar truth。
3. Desktop 前台只保留一个一级对象：`Manager`。
4. `Manager thread = 一个 mission 的连续前台表示`；`history` 只是这条线程的连续性和筛选视角，不再是独立 surface。
5. `supervisor / recovery / studio` 不再作为一级页面或一级对接对象；它们只作为同一条 Manager 对话中的模式、发言来源或可钻取子流。
6. Desktop 的默认姿态固定为“多藏”：非 Manager 的 agent 输出默认折叠、摘要化，并允许点击进入完整输出流。
7. Desktop 的核心资产应定义为：`Manager conversation UI + display protocol`；`manager / leader / team agents / templates / contracts / receipts / recovery` 仍归 runtime 侧。
8. 当前 shared surface 与 DTO 已经具备 `mission_console / task_contract_summary / latest_receipt_summary / latest_artifact_ref / recovery_state / governance_summary` 这些必要真源切片；下一轮前端主线不是再造数据层，而是把 renderer 从多页 workbench 壳收成单 Manager 连续线程壳。

## 今日实施结果

   - 左侧改成 `Manager Threads`
   - 右侧改成同一 mission 的单对话页
   - `Mission / Studio / Recovery` 改成同线程 lens，而不是一级导航
2. Desktop 的 shared DTO 与 mock/e2e 已对齐当前 truth chain：
   - renderer 直接消费 `task_contract_summary / mission_console / latest_receipt_summary / latest_artifact_ref / recovery_state / governance_summary`
   - Electron mock 与 renderer `vitest` 都已改成 manager-thread 口径

   - `mission_console.derived_responsibility_graph`
   - `governance_summary`

## 现状对齐

当前代码基线已经说明两件事：

1. 底层 truth chain 是对的：
   - `task_contract.json`
   - `receipts.jsonl`
   - `recovery_cursor.json`
   - `surface_meta / mission_console / derived_responsibility_graph`
2. 但可见 Desktop 壳还停在较早的 workbench 叙事：
   - `App.tsx` 仍按 `home / flow / manage`
   - `FlowRail.tsx` 仍把多页导航放在左侧
   - `WorkbenchShell.tsx` 与 `DetailDrawer.tsx` 仍偏多块面板结构

因此，下一轮 Desktop 更新的主线不是“补更多卡片”，而是：

- 左侧收成 `Manager thread` 列表
- 右侧收成单条 Manager 主对话
- 顶部只留 1-2 行必要任务信息与控制动作
- agent 子流默认折叠，通过点击进入完整输出
