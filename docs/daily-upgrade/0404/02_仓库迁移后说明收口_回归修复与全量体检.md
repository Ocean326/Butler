## 3. 回归修复裁决

### 3.1 chat / frontdoor / feedback

- `products/chat/negotiation.py` 修正迁移后 campaign 状态追问、`/plan` 显式模式、长任务自动开流和缺失 `mission_id` 时的 feedback 回退
- `products/chat/mainline.py` 把 `plan` 纳回显式前门模式识别
- `products/chat/feishu_bot/transport.py` 统一帮助文案，并把配置路径解析兼容到 canonical 树与裁剪测试工作区
- `products/chat/memory_runtime/recent_scope_paths.py` 补齐 compat legacy recent-memory 目录清理
- `butler_main/butler_cli.py` 的迁移提示改到现役 `butler-flow new ...`

### 3.2 runtime / orchestrator / skills

- `agents_os/skills/manager_agent.py` 改为从现役 `butler_main/platform/skills/` 读取 skill bundle
- `products/campaign_orchestrator/orchestrator/interfaces/observe.py` 改用 `sys.stdout.write`，避免 JSON CLI 输出被时间戳 hook 污染
- `tools/orchestrator_layer_audit.py` 与 `tools/runtime_os_codemod.py` 改到 canonical 路径口径
- `runtime_os` 顶层 compat 壳继续保留，但命名空间测试改为校验“导出身份 + 文件源一致”，不再把 import 顺序下的模块对象同一性当契约

### 3.3 测试与兼容面清理

- `orchestrator/smoke.py` 与 `test_orchestrator_smoke.py` 对齐当前 `agent_turn` 主线语义
- `test_legacy_execution_shell_tombstones.py` 接受 legacy 目录已彻底移除的收口态
- 已退役的 `test_self_mind_services.py` 从默认回归移出，迁到 `tests/manual/obsolete/`
- 桌面端 Electron e2e 修复为每例独立 user-data/home 目录，避免 `localStorage` 串测导致第二例卡在 `Config Path Fallback`
