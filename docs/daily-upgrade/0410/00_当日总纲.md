# 0410 当日总纲

日期：2026-04-10
状态：已落代码 / 已验证 / 当前真源
所属层级：Product Surface（主） / L1 `Agent Execution Runtime`（辅）

## 今日主线

1. 暂时关掉 Butler Desktop 当前前端壳里和这轮目标无关的模式与面板，不再同时追求 runtime、studio、agent detail 等完整工作台叙事。
2. 把 Desktop 前台壳进一步收窄成一个适合增量开发的最小 `Manager shell`：
   - 左侧只保留品牌、`New thread`、真实 history cards
   - 右侧只保留 `Manager` 主对话、消息流和 composer
3. 在不改变 shared surface truth owner 的前提下，为 Desktop 新增最小 manager message 流式 IPC，让右侧主对话具备真实 streaming 感。
4. 保留现有黑/日模式，不把这轮视觉收口做成一次性大改色或新品牌系统。

## 一句话裁决

从今天起，Butler Desktop 当前前台壳的默认开发起点收口为：

> **先用最小 `Manager conversation shell` 建立稳定骨架；其他模式先退后，不抢第一层产品成立。**

补充裁决：

- 左 rail 当前只承担 thread continuity，不再承担产品模式导航。
- 右侧当前只认 `Manager` 为一级对话对象；runtime/studio/agent 先退出主路径。
- `sendManagerMessage` 之外，Desktop 现役新增 `manager-message-stream` IPC，用于右侧主对话的最小流式输出。

## 今日新增结论

1. 这轮 renderer 的正确动作不是继续叠加更完整的 workbench，而是先把现有壳临时收窄。
2. 左侧现役只保留：
   - 品牌区
   - `New thread`
   - 真实 manager thread history cards
3. 右侧现役只保留：
   - 极薄 `Manager conversation` header
   - 基于 manager blocks 组装的消息流
   - 底部 composer
4. 当前无关主路径的 UI 先退出首屏与默认 render tree：
   - `Runtime / Studio` 模式切换
   - snapshot cards
   - runtime/studio strip
   - agent detail sheet
   - code split pane
5. Desktop 继续只经由 `preload + IPC + desktop bridge` 消费 shared surface；这轮没有新增 Desktop 私有 truth。
6. 当前流式对话通过 `manager-message-stream` IPC 实现，允许以最小 started/chunk/completed/failed 事件把最终 response 切片后逐段送入 renderer。

## 今日完成结果

1. renderer 现已收口成双栏最小骨架：
   - 左 rail
   - 右侧 manager conversation
2. `MissionShell` 已从多模式 workbench 壳改成更克制的 conversation shell。
3. `App.tsx` 已改成最小状态容器，只保留：
   - config attach
   - thread history 切换
   - new thread
   - manager stream send
4. `mission-shell.ts` 已新增 conversation message 归一化 helper，用于把 manager blocks 收成 chat-like 消息流。
5. Desktop 新增流式 IPC：
   - `sendManagerMessageStream`
   - `onManagerMessageEvent`
   - `manager-message-stream` channel/event
6. renderer tests 已更新到新的最小壳口径，并覆盖真实 stream mock。
