## 今日裁决

1. 公开命令面收口为：
   - `butler-flow`
   - `butler-flow new`
   - `butler-flow resume`
   - `butler-flow exec`
2. `run` / `exec run` 作为兼容别名保留一版，但不再出现在首屏帮助或 launcher 主动作中。
3. TTY 下 `new` 固定进入 setup picker；Textual 可用时走全屏向导，不可用时走 plain 向导。
4. setup picker 固定：
   - `goal`
   - `mode=single|flow`
   - `level=simple|medium|high(禁用)`
   - `flow=<builtin catalog | free | template:<id>>`
   - `guard`
   - `confirm`
5. `free` 的当前产品语义是“先创建 shared template，再按模板启动 flow”，不是直接落一个自由 instance。
6. 默认 home 固定是 `workspace`；旧 `/history` 只保留静默兼容别名，不再作为产品级主导航命令。
7. `workspace` 左侧只列 `instance` flows，按最近更新时间排序；右侧只展示 runtime preview / runtime timeline，不承担 shared asset 管理。
8. `single flow page` 固定是 runtime console；允许只读显示 instance definition / source template 摘要，但不在这里做 definition mutation。
9. `/manage` 固定只管理 shared flow assets：
   - `builtin`
   - `template`
10. `instance` definition 归单 flow/runtime 侧，不再进入 `/manage` 资产中心。
11. `/list` 打开 `/manage`；`/flows` 只保留迁移提示并跳转 `/manage`，不再作为产品级设计中心。
12. `single flow page` 固定重设为：`默认 supervisor 单栏流 + Shift+Tab 切到 workflow 流`。
13. `approval_state / latest judge / latest operator receipt / handoff / role / phase` 不再固定塞进独立 rail 或 inspector，而是优先通过结构化流式事件外显；`supervisor` 相关主信息并入流开头。
14. 多 agent 在主界面只保留紧凑主信息：`active role + execution/session mode + latest handoff summary`；完整 role/runtime 辅助信息先并入 `supervisor` 流开头，不再单独占右侧位置。
15. repo-local 资产树仍保留在 `butler_main/butler_bot_code/assets/flows/{builtin,templates,instances}`，但当前产品心智已分流：
   - shared assets：`builtin + template` -> `/manage`
   - runtime instances：`instance` -> `workspace + single flow`
