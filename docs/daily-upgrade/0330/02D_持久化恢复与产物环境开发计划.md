## 一句话裁决

`02D` 负责 `checkpoint / writeback / recovery / artifact-workspace linkage`。  
Butler 要吸收 DeerFlow、LangGraph 一类 harness 的长任务优势，关键不在 UI，而在这层能否把中断、恢复、产物和环境真正做成 durable fact。

## 当前已对齐能力

1. `FileWorkflowCheckpointStore` 已提供 checkpoint save/get/latest 与 `current.json` 回写。
2. execution runtime 已具备 blocked receipt 与 checkpoint resume 的最小闭环。
3. workflow 产物、run artifacts、session bundle 已开始形成基本文件布局。
4. `workflow_session_id`、`checkpoint_id`、receipt 已经可以形成基础恢复证据链。

## 当前缺口

1. `WritebackCoordinator` 仍偏 stub，外部持久化写回与 durable receipt 的一致性不够强。
2. `resume / retry / skip / force_transition` 尚未统一成同一恢复动作合同。
3. artifact、workspace、filesystem、bundle root 之间的边界还不够清楚，容易把“本地目录存在”误当成“恢复事实成立”。
4. replay beyond resume、time-travel、cleanup/retention 仍缺少正式口径。

## P0 开发计划

1. 冻结最小恢复动作合同，统一 `resume / retry / skip / force_transition` 的入参、回执和审计键。
2. 把 checkpoint、durability receipt、recovery decision、artifact linkage 串成同一条证据链。
3. 明确 `artifact / bundle root / workspace / filesystem env` 四者边界，避免本地环境与产品真源混层。
4. 为 `writeback` 建立最小真实落地点，避免“执行完了但 durable linkage 仍只是占位”。

## P1 开发计划

1. 增加 replay from checkpoint 的正式口径，不只停留在 resume。
2. 增加 cleanup/retention 策略，防止长期运行的 run/session 无上限膨胀。
3. 为 operator 触发的恢复动作补充更明确的 precondition/postcondition。
4. 把 thread-scoped filesystem 与 artifact registry 的连接做成稳定可诊断对象。

## P2 开发计划

1. 增加 time-travel、snapshot diff、postmortem replay。
2. 增加多来源恢复证据的归一化对照。
3. 增加 workspace template / sandbox profile 的版本化治理。

## 关键合同

1. `checkpoint`
  - 是可恢复状态快照，不是普通调试日志。
2. `durability receipt`
  - 用于说明一次写回、恢复或持久化动作是否真正落地。
3. `bundle root / artifact linkage`
  - 用于定位产物与会话/任务的关系，不用于表达产品态 summary。
4. `recovery action contract`
  - 必须同时包含触发原因、操作者、前状态、后状态、关联 receipt。

## 验收口径

1. `test_agents_os_runtime.py` 验证 checkpoint save/resume 与 blocked receipt。
2. `test_orchestrator_workflow_vm.py` 验证 recovery 和 step routing 没有破坏执行语义。
3. 若改动 campaign/query 外显恢复口径，补验 `test_orchestrator_campaign_service.py`。
4. 若改动文件布局或 namespace，补验 `test_runtime_os_root_package.py`。

## 明确不做

1. 不把 `recovery` 写成 orchestrator 私有散装逻辑。
2. 不把“磁盘上有文件”直接等同于“durable receipt 成立”。
3. 不把 time-travel 作为当前第一优先级，先保证 resume、writeback、artifact linkage 真实可验收。
