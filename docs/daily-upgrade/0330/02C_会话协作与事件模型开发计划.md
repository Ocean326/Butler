## 一句话裁决

`02C` 负责 `workflow session` 这一层的真对象。  
Butler 的 `session / mailbox / handoff / join / artifact registry / event log` 要先成为稳定会话语义，产品层和研究层才能共享同一套多 agent 运行边界。

## 本文边界

- 主层级：`L4 Multi-Agent Session Runtime`
- 次层级：`L3 Multi-Agent Protocol`、`Product Surface`
- 主代码目录：
  - `butler_main/runtime_os/multi_agent_runtime/`
  - `butler_main/runtime_os/process_runtime/session/`
  - `butler_main/multi_agents_os/session/`
  - `butler_main/orchestrator/runtime_bridge/workflow_session_bridge.py`
- 默认测试：
  - `test_multi_agents_os_factory.py`
  - `test_multi_agents_os_collaboration.py`
  - `test_runtime_os_namespace.py`
  - `test_orchestrator_campaign_observe.py`

## 当前已对齐能力

1. `FileWorkflowSessionStore`、`FileWorkflowEventLog` 已提供 session bundle 与 event log 的文件落地。
2. `ArtifactRegistry`、`WorkflowBlackboard`、`CollaborationSubstrate` 已提供 artifact、共享板、mailbox、handoff、ownership 的最小数据结构。
3. `OrchestratorWorkflowSessionBridge` 已进入 orchestrator 主链，具备 session projection 组装能力。
4. `workflow_session_id` 已经能作为 durable execution 的核心关联键之一。

## 当前缺口

1. `join_contract / mailbox_messages / blackboard` 更像结构化容器，还不是 engine 普遍消费的正式运行语义。
2. `thread / turn / item` 与 `workflow session` 的映射还没有在 Butler 内部冻结成统一事件模型。
3. `subagent / handoff / role binding / artifact visibility` 的 operator 可见合同仍偏薄，跨 surface 读取成本高。
4. research 路径和主线执行路径对 collaboration projection 的消费强度不一致，容易造成“同样有 session，读法却不同”。

## P0 开发计划

1. 冻结 `workflow session bundle` 的主对象边界：
   - session metadata
   - event log
   - artifact registry
   - mailbox
   - join contract
   - ownership
2. 定义 `thread / turn / item -> workflow session` 的映射口径，避免产品语义和 runtime 语义各说各话。
3. 把 `handoff` 定义成显式 session event，而不是散落在 message、trace、projection 三处的弱关联事实。
4. 冻结 artifact visibility 与 role binding 的最小规则，保证子代理与多角色协作有稳定可见性边界。

## P1 开发计划

1. 为 `join / barrier / mailbox trigger` 增加 engine 级消费规则。
2. 增加 session projection 给 console/query 的统一只读合同。
3. 统一 research 路径与主线路径对 collaboration projection 的生成与消费方式。
4. 为 `subagent lifecycle` 增加更明确的开始、阻塞、完成、回传事件。

## P2 开发计划

1. 为 A2A 或 remote agent adapter 预留 session event bridge。
2. 增加跨 session 的 thread lineage 与 parent/child session 关系。
3. 增加更细的 artifact lineage 与 join diagnostics。

## 关键合同

1. `workflow_session_id`
  - 是 L4 的稳定身份，不是产品层随意生成的 view id。
2. `session event log`
  - 承担会话过程真事实，不等于 trace spans，也不等于产品 summary。
3. `ArtifactRegistry`
  - 承担 artifact 可见性与引用事实，不承担文件系统实现细节。
4. `handoff / mailbox / join contract`
  - 是会话协作原语，不是 UI 特效或 research 私有结构。

## 验收口径

1. `test_multi_agents_os_factory.py` 验证 session bundle 创建、加载、持久化与回读。
2. `test_multi_agents_os_collaboration.py` 验证 artifact visibility、mailbox、handoff、join、ownership 的协作语义。
3. `test_runtime_os_namespace.py` 验证现役命名空间与兼容目录映射。
4. 若改动外显 projection，补验 `test_orchestrator_campaign_observe.py` 或 `test_console_server.py`。

## 明确不做

1. 不把 message swarm、临时聊天记录或 console 面板状态直接当作 L4 真源。
2. 不把研究路径的便捷结构默认扩散成主线合同。
3. 不把 `thread`、`turn`、`item` 直接等同于 `session`，四者必须保持层内映射关系而非同义。
