## 今日裁决

1. `session_scope_id` 仍是 Butler chat 连续性的主键；`chat_session_id` 是该 scope 内部的当前会话主线 id。
2. `chat_session_id` 必须由 Butler 自持，不依赖 vendor-native thread。
3. 当前 session 不命中时，frontdoor 首轮要 bootstrap 一个 `chat_session_id`，不能等到回复结束后才补。
4. router 要把 `router_session_action / router_session_confidence / router_session_reason_flags / chat_session_id` 一起写回 metadata。
5. recent memory、recent raw turns、summary pool 必须按 `chat_session_id` 过滤，只给当前内部 chat session 投喂上下文。
6. prompt 只增加轻量 `session_selection` 指示块：继续当前主线时补全省略；命中新话题时不要误续旧线；仅在多高概率解释并存时才澄清。
7. `project` sticky state 继续保留，但 review 态不应因为“实现方案”这类 review 语境词被误推进到 `imp`。
8. chat/frontdoor 的 CLI 默认改为 `cursor_fast` lane，不改全局 `cli_runtime.active=codex` 的系统默认。
9. frontdoor 对普通 chat turn 的最终 `cli/model` 选择归 mainline compile 所有；`chat/engine.py` 只保留用户显式 override 解析与信息查询，不再先拍板默认 CLI。
10. `cli_runner` 的 Codex-first 默认晋升不能覆盖 chat router 已明确选中的 lane。
11. `/manage` 继续只管 `builtin + template` shared assets，不接管 `instance` runtime。
12. shared asset 的真源继续是 `assets/flows/{builtin,templates}/*.json`，但现役定义升级为“JSON 合同 + companion bundle”。
13. manager agent 的现役生命周期升级为 `proposal / build / review / commit`。
14. builtin 资产每次修改都必须显式二选一：
    - `clone` 到 editable template
    - `edit` 原地修改 builtin
15. supervisor 认知采用“手写 + 编译混合”：bundle 内的 `supervisor.md` 与 `derived/supervisor_knowledge.json` 一起进入 runtime/compiler 注入面，而不是把完整 prompt 文件当唯一真源。
16. `/manage` 必须把 “manager chat” 与 “asset edit” 分离：自然语言会话不再复用 `manage_flow()` 的 builtin edit 路径，避免当前焦点 builtin 粘滞导致误触发 `clone/edit` 强校验。
17. manager chat 现役支持 Codex 同 session 续接；当 manager chat 正在运行时，新消息先入队，等待当前 manager turn 完成后续发。
18. `role_guidance` 是 shared asset 的建议型静态字段，不是强制团队制度；supervisor 仍是唯一外部主脑，必要时才按需调用 `creator / product-manager / user-simulator` 等临时角色。
19. supervisor stream 必须结构化外显 Input / Output / Decision；heuristic supervisor 也要补合成 input/output 记录。
20. transcript filter 新增 `supervisor`，不影响既有 `all/assistant/system/judge/operator`。
