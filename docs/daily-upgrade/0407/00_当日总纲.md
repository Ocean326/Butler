## 今日新增结论

1. `CLI agent` 是 `execution substrate`，不是 `organizational kernel`
2. `group chat / @ / 1:1 stream` 只能做 projection，不做真源
3. `caller != authority`，human/operator/manager/end-user 必须显式分帽子
4. `v1` 应按单一 proof slice 收窄，而不是先做平台切口
5. 当前更适合的名称是 `canonical team runtime over vendor substrate`
6. 若继续往 full-stack 远景推进，第一真源应立在 `task + artifact`，而不是 `team config` 或 `session/thread`
7. `上级 / 下级` 应保留为关键关系，但只能是 `ownership / authority / spawn` 图的一部分，不再作为唯一组织模型
8. 默认产品入口更适合 `Mission Board + Chat`，工作台负责投影，不负责改写真源
9. 当前 `v1` 只证明 `Receipt-backed Repo-Bound Task Contract Runtime`，不再同时证明完整 recursive team、portable semantics 或 general knowledge work
10. `butler-flow -> canonical team runtime` 的升级路径必须 `subtraction-first`：先冻结 truth owner，再补 contract / receipt / recovery / authority
11. 在 `P6` closure gate 通过前，`team graph` 只能是派生图，`Mission Console` 只能是 projection，不得越级升格成独立真源
12. `docs-only system` 继续按 `system` 严格回写，但默认直接并回当前分支 / `main`，不再自动创建 sibling worktree

## 本轮实现回写（`P0-P5` 已落地）

   - `task_contract.json`：任务真源
   - `receipts.jsonl`：accepted progress / governance ledger
   - `recovery_cursor.json`：transcript-independent recovery pointer
2. `flow_definition.json` 已明确降级为 contract materialization，当前保存：
   - `task_contract_id`
   - `task_contract_summary`
   - `truth_owner=task_contract.json`
   - launch/runtime defaults 与 bundle/source 输入
3. `workflow_state.json` 已继续降级为 runtime/recovery cache；合同、accepted progress 与恢复指针的权威写入已分别移到 `task_contract.json` / `receipts.jsonl` / `recovery_cursor.json`。
4. `runtime` 当前会把这些动作统一补写到 canonical receipt spine：
   - `ADVANCE / COMPLETE -> turn_acceptance`
   - accepted artifact -> `artifact_acceptance`
   - operator action -> `operator_action`
   - exec terminal -> `exec_terminal`
   - authority / policy change -> typed governance receipt
5. `new / resume / exec / status / workspace / single-flow` 当前已能围绕同一个 `task_contract_id` 读取 contract + receipt + recovery truth。
6. `resume` 已开始真实消费 recovery truth：
   - 可区分 `resume_existing_session`
   - `reseed_same_contract`
   - `rebind_role_session`
   - `rollback_to_receipt`
   - `pause_for_operator`
7. 旧实例继续采用懒迁移：读取 legacy `flow_state.json` 或缺失 `task_contract.json / receipts.jsonl / recovery_cursor.json` 的实例时，会自动 bootstrap 当前最小 truth slice。

   - authority/policy 变更会先写 typed governance receipt，再同步 `task_contract.json` 当前快照，并刷新 `recovery_cursor.json`

   - `status / workspace / single-flow / manage-center` 已稳定外显 `governance_summary / mission_console / surface_meta / derived_responsibility_graph`

9. provider / patch-path 兼容面也已同步硬化：
