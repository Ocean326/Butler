# 变更说明：心跳并行执行与分支可见性

- 日期：2026-03-08
- 范围：`butler_bot_code/butler_bot/memory_manager.py`、`butler_bot_code/tests/test_memory_manager_maintenance.py`、`butler_bot_code/docs/SELF_COGNITION_butler_bot.md`、`butler_bot_code/README.md`
- 目标：将心跳从单提示串行执行升级为受控并行 + 汇总，并补齐分支执行可见性与可追溯性。

## 背景

此前单轮心跳只执行一个 `execute_prompt`，在多个互不依赖任务并存时吞吐偏低；且缺少每轮并行方式与 defer 原因的结构化留痕，不利于在飞书侧排障和复盘。

## 功能变更

1. 心跳执行模型升级为 Agent Team 风格。
- 单轮支持 `task_groups/branches`。
- 组内可并行执行，默认并行上限 3 路（配置 `heartbeat.max_parallel_branches`，代码硬上限 3）。
- 复杂任务允许拆成 2-3 个独立分支，超预算任务通过 `deferred_task_ids` 延后。

2. 兼容旧 schema。
- 若规划器仅返回旧字段 `execute_prompt`，自动回退单分支执行路径，避免旧提示词失效。

3. 增强状态回写语义。
- 短期任务按分支结果回写 `done/deferred/last_result`。
- 长期任务仅在分支显式触达 `touch_long_task_ids` 时推进 `next_due_at`，不再因“仅到点”自动推进。

4. 新增分支可见性摘要（用户可见）。
- 每轮消息会附带：
  - 是否并行
  - 并行分支数/串行分支数
  - 分支成功/失败数
  - defer 数量与 defer 原因

5. 新增每轮快照持久化（recent_memory）。
- 每轮心跳都会写入一条 `topic=心跳执行快照` 的 recent entry。
- 扩展字段 `heartbeat_execution_snapshot`，含：
  - `parallel_used`、`parallel_branch_count`、`serial_branch_count`
  - `execution_mode`、`chosen_mode`
  - `deferred_task_ids`、`defer_reason`
  - 分支级信息（`branch_id`、`run_mode`、`ok`、`duration_seconds`、`output_preview`）

## 关键实现点

- 新增/重构方法：
  - `_execute_heartbeat_plan`
  - `_summarize_heartbeat_branch_results`
  - `_persist_heartbeat_snapshot_to_recent`
  - `_apply_heartbeat_plan`（分支级回写）
- 规划器 schema 扩展：`execution_mode`、`task_groups`、`deferred_task_ids`、`defer_reason`。

## 验证结果

已执行：

```powershell
python butler_bot_code/tests/test_memory_manager_maintenance.py
python butler_bot_code/tests/test_memory_manager_recent.py
```

结果：均通过（`OK`）。

新增验证覆盖：
- 并行上限 3 路生效。
- 快照写入 `recent_memory`。
- 消息中可见并行信息和 defer 信息。
- 长期任务未触达不推进 `next_due_at`。

## 已知边界

- 当前并行上限固定硬帽 3；如需更高并发，需要重新评估超时、模型配额和消息稳定性。
- 快照写入 recent 后会参与现有 compact 机制，超量时可能被压缩到归档摘要中。

## 回滚点

若需快速回滚，可优先回退以下逻辑（按影响面从小到大）：
1. `_persist_heartbeat_snapshot_to_recent`（仅可见性，不影响主执行）
2. `_summarize_heartbeat_branch_results` 中新增可见性文案
3. `_execute_heartbeat_plan` 并行路径（保留 legacy 单分支）

## 2026-03-08 优化（规划提示词、拆分逻辑、执行器与 sub-agent 约定）

- **规划提示词**（`_build_heartbeat_planning_prompt`）：新增第 10 条要求——任务拆分与 agent_role：每个 branch 的 `agent_role` 填逻辑角色之一（literature | file-manager | research-ops | secretary | engineering-tracker | discussion | orchestrator | governance），与 governance `Subagent_可调用清单与典型场景_20260308.md` 一致；branch 的 prompt 必须写清「你作为 XX-agent，执行…，产出写到 工作区/对应子目录/」；可并行判定与典型可并行组合（文献+文件、文献+研究思路等）写清，同组内无依赖分支 `can_run_parallel` 为 true。
- **工作区约定**（`HEARTBEAT_WORKSPACE_HINT`）：补充一句：若本分支对应某逻辑角色，请在任务开头写明「你作为 XX-agent，…」，产出写到工作区/对应子目录。
- **拆分逻辑**（`_normalize_plan_task_groups`）：单轮 `max_groups` 从 2 调整为 4，便于多组串行；执行器内注释明确「可并行：无 depends_on 且 can_run_parallel；其余串行，在并行完成后按原序执行」。
- **衔接约定**（`local_memory/心跳规划与执行衔接约定.md`）：新增 §2.1 执行器与 sub-agent 的关系——当前每支为单次 CLI 调用，agent_role 用于日志与 prompt 自描述；与飞书工作站 mcp_task 并行契约一致；后续可扩展为按 agent_role 路由到 subagent_type。
