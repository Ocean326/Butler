## 0321 为什么会显得很乱

1. 同一条 `Talk + AgentOS` 主线同时存在 `升级计划 / 总装 / 评审 / phase 结果 / 主线推进计划` 多份现役文档，容易把不同抽象层的状态混在一起。
2. `heartbeat`、`orchestrator`、`多 Agent 协作运行层 V0` 都在讲后台演进，但层级不同，昨天又没有最终裁线，所以很容易互相抢口径。
3. `research`、`orchestrator`、`多 Agent 协作运行层 V0` 都有真实进展，但抽象层不同，又都还没到“完整产品闭环”，所以很容易被误读成到处都只做了一半。

## 0322 的裁线结论

1. 今天唯一的前台硬主线是：跑通普通 `talk` 最小黄金路径，并把最终输出真正收口到 `FeishuDeliveryAdapter`。
2. 今天唯一的后台硬主线是：`orchestrator` 接管后台 runtime，不再继续在 heartbeat 上投入。
3. `多 Agent 协作运行层 V0` 今天作为架构副线保留，用来定义 `AgentRuntime` 与 `orchestrator` 之间的中间层，而不是取代 orchestrator。
4. 从现在开始，`0322` 目录下的 `02-04` 是今天的现役入口；`0321` 目录保留为背景时间轴。
