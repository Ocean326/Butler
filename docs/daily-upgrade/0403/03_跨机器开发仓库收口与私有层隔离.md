# 0403 跨机器开发仓库收口与私有层隔离

日期：2026-04-03  
状态：已实施  
所属层级：文档治理 / 仓库治理

## 本轮目标

把当前仓库收口成“可在另一台机器继续开发”的 core repo，同时把运行态实例、本机会话和个人私有配置拆回 machine-local overlay。

## 当前裁决

1. core repo 保留：
   - 代码
   - 正式文档
   - 精选研究资料
   - 可复用静态资产与模板
2. machine-local / runtime-local overlay 不再进 git：
   - `butler_main/butler_bot_code/assets/flows/instances/`
   - `butler_main/butler_bot_code/assets/flows/manage_sessions/`
   - `butler_main/chat/data/hot/recent_memory/` 的 live payload
   - `butler_main/chat/data/cold/local_memory/` 的私有层
   - `.codex/config.toml`
3. `butler_main/butler_flow/manager_prompt_assets/references/` 是当前 butler-flow manager 的静态依赖，必须进 git。
4. 跨机器开发的最小启动资料统一收口为：
   - `.env.example`
   - `.codex/config.template.toml`
   - `docs/concepts/跨机器开发仓库启动说明.md`
5. 研究层当前采用 split 策略：
   - `BrainStorm/Insights`、`Playbooks`、`Guides`、`Templates` 等继续允许进 git
   - `BrainStorm/Raw`、`BrainStorm/Working` 改为私有原始工作层

## 本轮改动

- 更新 `.gitignore`，明确 runtime / private overlay 边界
- 停止跟踪已入库的 flow instance、hot recent memory、`.codex/config.toml` 与 `local_memory` 私有层
- 抽出稳定研究稿 `docs/0402/20260402_KDD_Section1_en.md`，避免 `docs/0402` 继续依赖 flow-local artifact 路径
- 新增跨机器开发启动说明，并从 README / docs 导航层可达
- 把“repo portability / private overlay / runtime artifact boundary”补入 `project-map`
