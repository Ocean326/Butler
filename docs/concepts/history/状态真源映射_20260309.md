# Butler 状态真源映射（2026-03-09）

> 状态说明（2026-03-25）：本文主体记录的是旧状态真源设计，其中 `beat_recent_memory / heart_beat_memory / heartbeat_long_tasks` 等均属于历史兼容期术语。当前应优先以 `task_ledger`、按 `session_scope_id` 隔离的 recent memory、以及 `chat / orchestrator` 运行数据为准。

## 目标原则

每类状态只允许一个 machine-readable 真源，其他形式均为只读镜像或历史归档。

## 四类真源

| 状态类型 | 目标真源 | 说明 |
|---|---|---|
| talk_recent_memory | `butler_bot_agent/agents/recent_memory/recent_memory.json` | 仅用于短期对话续接 |
| beat_recent_memory | `butler_bot_agent/agents/recent_memory/beat_recent/recent_memory.json` | 仅用于 heartbeat / 连续思考续接 |
| semantic_memory | `butler_bot_agent/agents/local_memory/` 下分层长期知识文件 | L0 索引、L1 摘要、L2 详情，保留长期事实、偏好、约定 |
| task_ledger | `butler_bot_agent/agents/state/task_ledger.json` | 短期任务、长期任务、提醒、执行状态统一账本 |
| ops_telemetry | `butler_bot_code/run/` + 后续统一 telemetry 文件 | 仅用于运行观测与排障 |

## 现阶段兼容写入

当前系统仍保留以下旧状态文件作为兼容镜像：

- `butler_bot_agent/agents/recent_memory/heart_beat_memory.json`
- `butler_bot_agent/agents/local_memory/heartbeat_long_tasks.json`
- `butler_bot_agent/agents/local_memory/heartbeat_long_tasks.md`
- `butler_bot_agent/agents/local_memory/heartbeat_tasks.md`

其中 `beat_recent_memory` 为新增活跃池：heartbeat 的执行快照与连续思考上下文优先写入该池，不再混写到 talk recent。

本轮开始，heartbeat 的执行结果会优先落到 `task_ledger.json`，并反向同步旧 JSON 镜像，保证旧逻辑暂时可用。
