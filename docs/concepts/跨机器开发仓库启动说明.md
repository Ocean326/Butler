# 跨机器开发仓库启动说明

用途：把当前仓库作为“代码 + 正式文档 + 精选研究”的 core repo，在另一台机器上快速恢复开发环境。

## 1. Clone 后先做的事

1. 创建 Python 虚拟环境：`python -m venv .venv`
2. 安装基础依赖：`./.venv/bin/pip install -r requirements-dev.txt`
3. 若需要前台 `butler-flow` TUI，再安装：`./.venv/bin/pip install -r requirements-cli.txt`
4. 若需要系统级 `butler-flow` 命令，再运行：`./tools/install-butler-flow`
5. 若需要 console webapp，在 `butler_main/console/webapp/` 下执行：
   - `npm install`
   - `npm run build`

## 2. 配置文件

- Butler 运行配置模板：`butler_main/butler_bot_code/configs/butler_bot.json.example`
- 环境变量模板：`.env.example`
- 项目级 Codex 配置模板：`.codex/config.template.toml`

建议做法：

1. 复制 `butler_bot.json.example` 为机器本地配置
2. 按需把 `.env.example` 的变量写入本机 env 文件或 shell profile
3. 若要使用项目级 Codex 配置，复制 `.codex/config.template.toml` 为 `.codex/config.toml` 后再填写本机私有项

## 3. 哪些内容不再随 core repo 分发

- `butler_main/butler_bot_code/assets/flows/instances/`
- `butler_main/butler_bot_code/assets/flows/manage_sessions/`
- `butler_main/chat/data/hot/recent_memory/` 的 live payload
- `butler_main/chat/data/cold/local_memory/` 的私有层
- `.codex/config.toml`

这些内容应视为 machine-local / runtime-local overlay，而不是仓库真源。

## 4. 最小验证

```bash
./.venv/bin/python -m pytest butler_main/butler_bot_code/tests/test_butler_flow.py -q
./.venv/bin/python -m pytest butler_main/butler_bot_code/tests/test_chat_recent_memory_runtime.py -q
./.venv/bin/python -m pytest butler_main/butler_bot_code/tests/test_chat_cli_runner.py -q
```

如需前台入口验证，再运行：

```bash
butler-flow preflight
```
