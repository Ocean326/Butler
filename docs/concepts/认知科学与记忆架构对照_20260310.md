# 认知科学与 Butler 记忆架构对照

更新时间：2026-03-10

补充说明：如果你现在关心的是“`self_mind` 到底算不算记忆、它和 `local memory` 的边界是什么、怎样让 Butler 从有知识走到有回忆”，请优先连读 `./self_mind与回忆机制_总体设计_20260311.md`。本文更偏认知概念对照；新文档更偏 Butler 当前语义层 / 回忆层 / 主意识连续层的总设计。

本文档对照认知科学中与记忆、意识、巩固相关的经典概念，梳理 Butler 当前实现的对应关系，并标出可借鉴或缺失的部分，便于后续设计有据可依。

## 一、记忆系统划分（认知心理学 / 神经科学）

### 1.1 经典划分

| 认知科学概念 | 简要说明 | Butler 中的对应 |
|-------------|----------|-----------------|
| **工作记忆 (Working Memory)** | 容量有限、当前任务相关的临时保持与操作；与中央执行、语音环、视空间画板等子成分相关。 | **决策器 + 当前轮 candidate**：决定是否触发潜意识、是否写 long-term；单轮内的「当前在做什么」由 planner/executor 的 context 承担。 |
| **短期/近期记忆 (Short-term / Recent)** | 秒～分钟级保持，易受干扰；未经巩固则遗忘。 | **unified recent**（`recent_memory.json`）：talk / mental / relationship_signal / task_signal / heartbeat_observation 等多流统一 schema，带时间戳与 scope。 |
| **长期记忆 (Long-term Memory)** | 持久存储；常分为陈述性（情节+语义）与程序性（内隐）。 | **local memory**：L1 Markdown + L2 detail + **L0 index**（当前结论、历史演化、适用情景），供 heartbeat 与后续检索。 |
| **情节记忆 (Episodic)** | “何时何地发生了什么”的具体经历，可回忆细节与情境。 | recent 中的 **topic / summary / event_type / raw_user_prompt** 及 companion 的 mental/relationship/task 流；L2 detail 中的长文本。 |
| **语义记忆 (Semantic)** | 抽象事实、概念、关系，脱离具体情境。 | L0 的 **current_conclusion / applicable_scenarios / keywords**；Soul/role 中的稳定描述。 |
| **程序性/内隐记忆 (Procedural / Implicit)** | 无需意识提取的“怎么做”（技能、习惯）。 | 当前**未显式建模**；可视为 heartbeat 的「执行习惯」、技能调用模式等，尚未以独立记忆类型存在。 |

### 1.2 陈述性 vs 程序性

- **陈述性 (Declarative / 外显)**：可陈述的“是什么/发生过什么”，对应你的 recent + local memory 的语义化条目。
- **程序性 (Procedural / 内隐)**：难以言说的“如何做”，对应认知科学中的技能与习惯。  
- **项目缺口**：没有单独的「程序性记忆」槽位或索引；若希望 Butler 显式区分“用户教过的方法/流程”与“事实与偏好”，可考虑为长期记忆增加类型标签（如 `memory_type: semantic | episodic | procedural`）或独立 procedural 流。

## 二、记忆巩固 (Memory Consolidation)

### 2.1 巩固类型（神经科学常用框架）

| 概念 | 说明 | 与 Butler 的对应 |
|------|------|------------------|
| **突触巩固 (Synaptic consolidation)** | 学习后数小时内，突触层面的强化（如 LTP）。 | 可类比为 **单轮内 subconscious 的整理**：从 candidate → primary_entry + companion_entries + trigger_level，即“当轮写入前的稳定化”。 |
| **系统巩固 (Systems consolidation)** | 数周～数年，从海马依赖的表征逐渐迁移到新皮层，形成更稳定、抽象的表征。 | **recent → local memory (L0/L1/L2)**：从多流 recent 提升 long_term_candidate，再写入 L1/L2 并更新 L0 索引；L0 的「当前结论 + 适用情景」接近“脱离具体情境的语义化”。 |
| **再巩固 (Reconsolidation)** | 提取会暂时使记忆变得可塑，可被更新后再稳定。 | 文档中「历史演化」与**更新已有 local memory** 的机制；当前实现是启发式文本（`_build_history_evolution`），**尚未做成“检索→进入可塑窗口→写回”的显式再巩固流程**。 |

### 2.2 睡眠/离线巩固（与 heartbeat 的类比）

认知科学中，**睡眠期间**海马与新皮层通过 sharp wave-ripples、delta、spindle 等节律协同，完成“近期经历 → 长期语义/图式”的迁移与抽象。

| 科学概念 | Butler 类比 |
|----------|-------------|
| 离线、非对话时段处理 | **Heartbeat**：在无用户对话时运行，读 Soul/role/unified recent/local index，执行计划并产生 beat recent。 |
| 重放 (replay) | 当前 heartbeat 读的是「已有 recent + L0 索引」，尚未实现“对近期经历的主动重放与优先巩固”（例如按 salience/importance 对 recent 做离线重算或摘要）。 |
| 抽象化、要点化 | L0 的 **current_conclusion**、L1 的「当前结论」可视为从情节到语义的抽象；**历史演化**若做成版本链，会更贴近“同一主题多次再巩固”的轨迹。 |

**可增强点**：

- 为 heartbeat 增加「离线巩固」步骤：在无紧急任务时，对高 salience 或带 long_term_candidate 的 recent 做批量回顾与摘要，或写入/更新 local memory。
- 显式区分「首次写入长期」与「对已有条目的再巩固更新」，并在 L0 或 L2 中记录 reconsolidation 来源（如 `updated_from_recent_id`、`reconsolidation_at`）。

## 三、显著性、注意与提取 (Salience, Attention, Retrieval)

### 3.1 显著性 (Salience)

认知神经科学中，**显著性**影响编码强度与后续提取概率；高显著性经验会增强前后相邻信息的记忆（retroactive / proactive enhancement）。

| 你的实现 | 认知对应 |
|----------|----------|
| **salience** 字段（0–1 浮点） | 编码/巩固权重的代理变量；companion 的 salience 在 primary 基础上微调（+0.05 / +0.1）。 |
| **confidence** | 可对应“记忆强度的主观/客观估计”，影响是否优先参与检索或巩固。 |
| **trigger_level** | 可视为“是否需要在本轮启动巩固流程”的离散决策，与“是否分配更多巩固资源”的注意/资源分配类似。 |

**可增强点**：

- 为 **retroactive / proactive enhancement** 建模：当某条 recent 的 salience 特别高时，是否对前一条或后一条 recent 做加权或延长保留时间。
- 将 salience 纳入「是否写入长期」「是否在 heartbeat 中被优先回顾」的显式条件（目前 long_term_candidate 更多由 hint 词与 relationship/mental 信号驱动）。

### 3.2 注意与工作记忆

- 认知上，**注意**在工作记忆与长期记忆中的机制可分离：工作记忆更依赖当前焦点与眼动/复述，长期记忆的注意更偏“基于线索的激活”。
- Butler 中 **active_window**（current / recent）可类比为“当前焦点 vs 近期可检索”；planner 的 context 组装（Soul + role + recent + L0）即“按线索激活的检索集”。

## 四、图式与知识结构 (Schema)

### 4.1 图式的作用

- **图式**：基于以往经验形成的知识结构，指导对新信息的理解、编码与整合。
- 与新信息**一致**时，更容易被整合进既有图式（schema-congruent）；**不一致**时可能触发新图式或图式修订。
- 前额叶（如 mPFC）在图式整合中起关键作用；海马保持细节，新皮层逐渐形成更概括的图式。

| 你的实现 | 图式角度的理解 |
|----------|----------------|
| **L0：current_conclusion + applicable_scenarios + history_evolution** | “当前结论”类似图式的当前状态；“适用情景”类似图式的激活条件；“历史演化”类似图式的修订轨迹。 |
| **keywords / context_tags** | 可作为图式的索引与检索键，决定“何时激活这条长期记忆”。 |
| **Soul + role** | 稳定的自我/角色图式，约束“Butler 是谁、默认如何行动”。 |

**可增强点**：

- **Schema-congruent vs incongruent**：对与现有 L0 结论高度一致的新信息做“合并/更新”，对明显冲突的做“新结论 + 历史演化”的显式分支，避免过度覆盖。
- **图式过强风险**：文献提示过强图式会导致错误记忆或刻板预测；可在 L0 或 prompt 中加入“仅在适用情景内信任该结论”“注意相反证据”的元信息。

## 五、自我与自传体记忆 (Self & Autobiographical Memory)

### 5.1 Self-Memory System (SMS)

- **Working Self**：当前活跃的目标与自我意象，调制对长期自传体知识的访问。
- **长期自传体知识**：约束“我是谁、经历过什么、能做什么”。

| 你的实现 | SMS 类比 |
|----------|----------|
| **Soul 摘录** | 稳定的“自我”知识（价值观、原则）。 |
| **Role 摘录** | 当前角色下的目标与行为约束（类似 working self 的角色面）。 |
| **recent + local memory** | 自传体情节（recent）+ 语义化自传体知识（L0/L1）。 |
| **relationship_signals / relation_signal** | 自我–他人关系与情绪，是自传体记忆的重要维度。 |

**可增强点**：

- **自我参照效应 (Self-Reference Effect)**：与自我相关的信息编码更强；可在 salience 或 long_term_candidate 的权重中，对“与 Soul/role/用户关系”明显相关的内容加分。
- **Source memory**：区分“用户说的”“Butler 推断的”“心跳执行的”；当前有 `derived_from`、`source_type`/`source_reason`，可进一步在 L0 或检索时区分来源可信度。

## 六、潜意识 / 内隐处理 (Subconscious / Implicit)

### 6.1 概念对应

- **内隐记忆**：无意识提取，却能影响行为（如习惯、启动效应）。
- **潜意识**在认知科学中常指：不被报告为“正在想”却影响判断与记忆的过程。

| 你的实现 | 对应关系 |
|----------|----------|
| **SubconsciousConsolidationService** | 不直接与用户对话，而是在每轮“背后”做整合：主 entry + companion + 隐式 long_term_candidate + trigger_level。 |
| **隐式提升 long_term_candidate** | 基于偏好词、relationship/mental、recent companion pattern 等信号，无需用户显式说“记下来”，类似“无意识层面识别出应巩固的内容”。 |
| **companion entries（mental / relationship_signal / task_signal）** | 将一轮对话中的不同“通道”拆开，部分对应可报告与不可报告的心理活动。 |

**可增强点**：

- **程序性/习惯**：若引入“用户常在某情境下希望 Butler 做 X”的统计或模式，可视为内隐的程序性记忆，与显式的 local memory 并存。
- **冲突与抑制**：认知中潜意识会参与冲突检测与抑制；Butler 可对“与 Soul/既有结论冲突”的新输入做显式标记或二次确认，而不是静默覆盖。

## 七、对照小结：已有 vs 可补全

### 7.1 已有且与认知概念对应良好的部分

- 统一 **短期多流**（talk / mental / relationship / task / heartbeat）→ 工作记忆/近期记忆的多通道。
- **潜意识整理**（主 entry + companion + 隐式 long-term 提升）→ 巩固前的稳定化与候选选择。
- **recent → local memory (L0/L1/L2)** → 系统巩固与情节→语义的抽象。
- **Soul + role + recent + L0** 作为 planner 上下文 → 自我图式 + 自传体知识 + 当前焦点。
- **salience / confidence / trigger_level / active_window** → 显著性、强度与注意/巩固决策的代理。
- **当前结论 + 历史演化 + 适用情景** → 图式式长期表征的雏形。

### 7.2 可补全或强化的方向

| 认知概念 | 建议 |
|----------|------|
| **程序性/内隐记忆** | 为长期记忆或行为模式增加类型（如 procedural），或单独“习惯/偏好模式”存储，供 heartbeat 与对话策略使用。 |
| **再巩固 (Reconsolidation)** | 将「历史演化」从启发式升级为版本链；对“检索到的 local memory 被本轮新信息更新”做显式流程与时间戳。 |
| **睡眠/离线巩固** | Heartbeat 中增加“离线巩固”步骤：按 salience/importance 回顾 recent，生成或更新 local memory，甚至显式 replay 高权重条目。 |
| **显著性驱动的增强** | 用 salience 参与 long_term_candidate 与离线巩固的优先级；可选：对高 salience 前后条目的 retroactive/proactive 增强。 |
| **图式一致/冲突** | 写入或更新 L0 时区分 schema-congruent（合并）与 schema-incongruent（新结论+演化）；避免图式过强导致的僵化。 |
| **自我参照与来源** | 对与 Soul/role/关系强相关的内容提高巩固权重；在 L0 或检索结果中保留并利用 source 信息（用户/推断/心跳）。 |
| **冲突与抑制** | 对新输入与既有结论/Soul 的冲突做检测与策略（如标记、二次确认、单独存为“待核实”）。 |

## 八、信息源可信度层级与时间衰减（科学表述）

### 8.1 你提出的原则（用更科学的说法）

你描述的是：**「当前连续的思维/动作」可信度最高，其次是经验、长期记忆和说明文档，且越旧可信度越低。**

在认知科学与记忆研究里，可以更精确地表述为两条互补原则：

1. **信息源层级（按“与当前情境的接近程度”）：**  
   当前正在进行的知觉/思维/动作（相当于工作记忆中的“在线”内容）→ 近期经验（近期情节记忆）→ 已巩固的长期记忆 → 外部文档/陈述性说明。  
   越“在线”、越未经存储-提取链路的，在**作为当前决策依据**时通常越可信，因为尚未经历编码失真、遗忘与再巩固中的扭曲。

2. **时间衰减（同一类来源内部）：**  
   在同类来源中，**越新（越近）的信息**往往越能反映“当前世界状态”和“当前用户状态”，因此作为行为依据时可信度更高；越旧的信息越可能过时或已被再巩固改写，需打折扣或与更新信息冲突时优先采信新的。

下面给出与这两条对应的科学概念与依据，以及和 Butler 实现的对应关系。

### 8.2 科学依据简述

| 角度 | 科学概念 / 现象 | 与“当前最高、越旧越低”的关系 |
|------|------------------|------------------------------|
| **遗忘与时间** | **艾宾浩斯遗忘曲线 (Ebbinghaus forgetting curve)** | 未经复习时，记忆保持量随时间单调下降；越久远的内容保留越少、失真越多，作为“事实依据”的可信度自然下降。[1] |
| **近因与提取** | **近因效应 (recency effect)** | 在情节记忆提取中，最近发生的内容更容易被正确、快速提取；近期信息在检索时更可用、也往往更准确。[2] |
| **来源监控** | **Source monitoring** | 人们会区分信息来自“自我生成/当前知觉”还是“外部/过去存储”。当前正在进行的思维与动作未经“存储→再提取”，没有经历来源混淆与提取诱发失真，在决策时更可依赖。[3] |
| **再巩固与失真** | **Reconsolidation** | 每次提取都会让记忆暂时进入可塑状态，可能被新信息更新或扭曲。因此“被提取次数多、时间久”的长期记忆，可能已多次再巩固，与最初编码时的状态不一致；越旧越可能被后续经验“改写”。[4] |
| **世界状态的时效性** | **Temporal validity / 信息陈旧** | 对“当前世界/用户状态”而言，文档、旧结论、旧偏好都可能已过时；决策应优先依赖当前对话、当前任务流和最新经验，其次才是历史结论与文档。[5] |

**小结（可写入设计文档的表述）：**  
在认知科学上，支持“**当前连续思维/动作的可信度最高，其次为近期经验、长期记忆与说明文档，且同一来源内越旧可信度越低**”的依据主要包括：  
(1) 遗忘曲线与近因效应（时间越久，记忆保持与准确性下降）；  
(2) 来源监控（当前在线信息未经存储-提取，失真更少）；  
(3) 再巩固（旧记忆可能已被多次更新或扭曲）；  
(4) 信息时效性（旧信息可能不再反映当前状态）。  

注意：哲学上“直接观察一定比存储知识更可靠”是有争议的；这里强调的是**在行为决策与状态估计时**，优先信任“当前情境流”和“更新鲜的经验/记忆”作为工程上的稳妥策略，而不是主张观察在认识论上绝对优先。

### 8.3 与 Butler 实现的对应

| 层级（从高到低） | 认知/系统含义 | Butler 中的对应 |
|------------------|---------------|------------------|
| **当前连续的思维/动作** | 工作记忆中“在线”的当前轮输入、模型输出、正在执行的动作与中间推理。 | 当前轮 `user_prompt` / `assistant_reply`、本轮的 candidate、heartbeat 执行中的 observation；未写入 recent 前的“当前上下文”。 |
| **近期经验** | 未经长期巩固的近期情节记忆；易提取、相对新鲜。 | **unified recent**（`recent_memory.json`）：talk / mental / relationship_signal / task_signal / heartbeat_observation 等；带时间戳，可按时间或 salience 排序。 |
| **长期记忆** | 已巩固的语义/图式化结论与历史演化。 | **local memory** L0 索引（current_conclusion、applicable_scenarios、history_evolution）及 L1/L2；检索时建议带“最后更新时间”或版本。 |
| **说明文档** | 外部、静态的陈述性知识；不随对话与行为自动更新。 | Soul、role 文件、工作区文档、外部说明等；作为稳定约束或参考，但与“当前状态”可能脱节。 |

**工程建议：**

- 在 **planner / 决策器** 中显式实现“信息源优先级”：冲突时优先采信当前轮与 recent，其次 local memory，再其次文档；并在 prompt 或配置中写清该顺序。
- 对 **local memory 与文档** 增加时间或版本信息（如 `last_updated`、`written_at`），在检索或冲突解决时实施“越旧权重越低”或“新旧冲突时以新为准”的策略。
- 将上述原则写入长期手册（如 Butler 认知记忆与 AI 演化手册），作为“证据层级”的固定约定，避免只靠模型自由发挥。
