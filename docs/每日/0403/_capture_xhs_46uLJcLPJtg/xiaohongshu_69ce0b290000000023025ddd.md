## Content

🔍 这两天在技术圈传得比较广的一篇长文，腾讯程序员 rickyshou 在腾讯技术工程官方号发表的 Agentic Engineering 实战记录。不是概念科普，是带着所有弯路和修正的两个月工程日志，从一个 AGENTS.md 文件长成 22 个 Agent、27 个 Skill 的完整体系。
	
📐 文章反复强调：上下文空间是 Agent 的认知带宽，任何你引入的管理机制都在消耗这个带宽。这种约束一旦确立，所有设计决策都有了锚点。作者用 DDD 设计了七版 Agent 体系，全部失败，最后领悟到 Agent 的接口不是函数签名而是上下文组合。于是重新设计了整套 agentic 系统，其中的工具调用理念跟我前几天发的 manus 方法非常接近。
	
🛠️ 文章部分要点：
	
1️⃣ 文档即记忆：人和 AI 读同一份 Markdown，高频使用天然对抗腐化，clone 即获得全部知识资产。不需要向量数据库，git 就是最好的知识管理系统
	
2️⃣ 三级工具体系按上下文影响分层：Command（极小入口）、Skill（可控知识包）、Subagent（隔离上下文）。分层依据是占用多少认知带宽，跟复杂度无关
	
3️⃣ 每次 AI 犯错花 2 分钟记录，这是复利的核心燃料。一个月后知识目录就初具规模，第二个月开始加速
	
💡 文章里五个独立实践（Anthropic、OpenAI Codex、everything-claude-code、Cursor、作者自己）收敛到同一组设计原则，这个多方印证本身就说明问题。建议边读边对照自己项目的 context 管理方式，会发现很多原来大家都踩过同一个坑的时刻。
