# 文档生命周期

更新时间：2026-03-26  
状态：现役  
用途：统一文档状态、迁移条件和更新触发规则

## 文档状态

- `现役`：当前可直接作为改动依据
- `稳定合同`：较长期稳定，默认优先读，例如 `docs/runtime/`
- `兼容期`：仍有参考价值，但术语或边界已部分过时，阅读时必须先看替代入口
- `历史`：只做追溯和对照，不能直接作为当前设计依据

## 目录职责

- `docs/project-map/`：当前导航层和改前读包
- `docs/daily-upgrade/`：时间线、推进记录、当日裁决
- `docs/runtime/`：稳定合同和复用接入说明
- `docs/concepts/`：长期原则、仍有效概念、接入说明
- `docs/concepts/history/`：历史设计稿、旧术语快照、退役方案

## 何时更新 `project-map`

满足任一条件，当天必须同步更新对应条目：

1. 当前系统默认术语变化
2. 分层边界变化
3. 某功能的主真源变化
4. 某功能的默认代码目录或测试入口变化
5. 某文档从现役降级为兼容期或历史

## 何时把内容从 `daily-upgrade/` 提升出来

满足下面两条时，优先提升到 `project-map/` 或 `runtime/`：

1. 连续多次被引用
2. 已经不再是“当日判断”，而是当前默认规则

## 历史迁移规则

满足任一条件可迁入 `docs/concepts/history/`：

1. 文档主体描述的是已退役结构
2. 当前文档顶部已经明确标注为历史快照
3. 当前实现和术语已被更新入口替代

迁移时要求：

1. 保留历史正文
2. 原路径留下跳转桩
3. 跳转桩必须写明替代入口和“不可直接作为改动依据”

## 周期维护

- 每次功能改动后：只判断是否要更新 `project-map`
- 每周：巡检一次 `docs/README.md` 当前入口、`concepts/` 状态、`daily-upgrade` 提升候选
