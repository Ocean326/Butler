## 核心论点（3 条）

### 1. 人类角色从"亲手做实验"迁移到"设计实验竞技场"

Autoresearch 的核心范式：人类只给高层目标（如"让模型更聪明"），系统每 5 分钟自动发起一轮实验，自动保留好方案、丢弃差方案，形成持续进化。真正的人类价值在于**定义规则、约束、奖励信号和搜索空间**，而非亲手跑每一次训练。

### 2. AI 在受限搜索空间内可以极快地"重新发现"人类多年成果

在约 17 小时的自动实验中，AI 重新发现了人类 8 年间逐步发展出的若干技术（如 RMSNorm）。这不是凭空创造，而是说明：**给定规则和搜索空间后，AI 可以高效遍历组合，找到与人类类似的优解**。实测从基线 1.506 优化到 0.975（约 35% 提升），且未触及架构性能上限。

### 3. "竞技场"思维可跨领域迁移

作者明确指出这套范式不局限于模型训练——在产品迭代、营销策略、增长实验等场景，一样可以通过"搭建自动试验场 → 让 AI 在规则内做 A/B 测试与策略搜索"来完成。核心抽象：**目标 + 规则 + 搜索空间 + 自动评估 = 自主进化循环**。

## 可借鉴点

| autoresearch 机制 | 抽象原理 | Butler 可映射方向 |
|---|---|---|
| 5 分钟自动实验周期 | 高频、低成本试错循环 | heartbeat 本身已是周期性循环，可在其中嵌入"micro-experiment"逻辑 |
| 自动保留好方案 / 丢弃差方案 | 基于评分的自然选择 | skill/配置的 A/B 评估：同一任务用两种 prompt 策略跑，保留效果好的 |
| 人类只设目标和搜索空间 | 高层目标与底层探索解耦 | Butler 用户只需表达意图，Butler 在受限工具空间内自主寻找最优执行路径 |
| 630 行极简核心 | 最小可行框架，可 fork 可改 | 提醒 Butler 核心代码保持精简；复杂度应在 skill/配置层，不在核心循环 |
| `results.tsv` 结果留痕 | 所有实验结果可回溯 | 与 task_ledger 理念一致——每轮执行结果必须持久化、可检索 |

## 与 Butler 当前架构的映射

| autoresearch 概念 | Butler 对应 | 差距 / 演进方向 |
|---|---|---|
| 实验竞技场 | heartbeat 执行环境 | heartbeat 目前是任务执行器，尚未有"自主探索最优策略"的 meta 层 |
| 自动评估函数 | task_ledger 完成判据 | 判据目前由 planner 人工写入，缺少自动化的"结果打分 → 策略保留"闭环 |
| 搜索空间定义 | skills 注册 + 工具白名单 | 已有边界约束，但尚未被当作"可调参的搜索空间"来治理 |
| 进化飞轮 | BrainStorm Insights + 经验沉淀 | 知识沉淀已在做，但缺少"自动从历史成功中提取策略并回注 prompt"的机制 |

## 行动建议（2 条）

1. **在 heartbeat 中试点 "micro-experiment" 机制**：对于非关键路径的重复性任务（如 prompt 组装策略、skill 选择），允许 heartbeat 在受控范围内自动尝试 2-3 种变体，记录效果差异，逐步积累最优策略。不需要复杂框架，只需在 task_ledger 中增加 `variant` 和 `score` 字段。

2. **将"竞技场"思维应用于 BrainStorm 流水线**：当前 Raw → Insight 的提炼完全依赖单次执行。可以设计一个轻量评估标准（信息密度、与 Butler 相关度、可操作性），让 heartbeat 自动给 Raw 条目打分排序，优先处理高分条目，减少人工选择成本。
