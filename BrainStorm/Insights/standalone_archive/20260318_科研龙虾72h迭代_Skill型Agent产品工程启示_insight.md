## 核心观点

### 1. Skill 规模与质量管控是 Agent 产品的核心竞争力门槛

科研龙虾上线时携带 431 个学术技能（skills），但三天内主动清理了 57 个质量不达标的。这揭示了一个关键工程规律：

- **Skill 的价值不在于数量，而在于可信度**：用户能否信任 agent 的每一次操作，取决于底层 skill 是否经过人工遴选和质量把关。
- **Skill 质量的退化是持续风险**：上线后暴露的问题（标签数量不对、暗色主题 UI 异常、删除后标签未同步）本质上都是 skill 内部逻辑或 skill 间状态一致性的缺陷。
- **"人工遴选"不是反模式，而是当前阶段的必要保障**：作者特意强调 431 个 skill "每一个都经过人工遴选"，这不是效率低下，而是在 AI 生成 skill 的时代，建立用户信任的唯一可信路径。

### 2. 测试覆盖率是 Agent 可信度的基础设施

三天内测试用例从 318 个涨到 1029 个，覆盖文献库操作、任务状态变更、文件写入等每一个原子操作。这指向一个被多数 agent 项目忽略的基本事实：

- **Agent 管理用户的高价值数据（文献、笔记、任务），容错空间极小**：不同于聊天机器人，操作型 agent 的一次错误可能导致数据丢失或污染，用户心理阈值远低于对话场景。
- **自动化测试是从"能 demo"到"可日用"的分水岭**：318 → 1029 的增长不是锦上添花，而是用户信任建立的前提。
- **测试的粒度应该对齐到 skill 级别**：每个 skill 的每个操作路径都应该有对应的自动化验证，而不是只测"端到端场景"。

### 3. Git 级文件变更追踪是操作型 Agent 的安全网

"所有工作区文件变更自动 Git 提交，误操作可以随时回滚"——这不是一个 feature，而是操作型 agent 必须内建的安全底线：

- **Agent 执行的每一次文件写操作都是不可逆风险**，尤其在管理用户的文献库、笔记和任务数据时。
- **自动版本控制将 agent 的操作从"不可逆"降级为"可回滚"**，本质上是用工程手段把 agent 的风险等级从"危险"降到"可控"。
- **这与 Harness 理念中"安全门控层"的思路一致**：不是阻止 agent 行动，而是确保行动有后悔药。

### 4. 用户反馈驱动的 72 小时高密度迭代暴露了 Agent 产品的特殊节奏

三天内修复大量 bug + 新增 16 个功能 + 测试用例翻三倍，这种节奏揭示了 agent 产品与传统 SaaS 的关键差异：

- **Agent 产品的"上线"不是终点而是压力测试的起点**：用户在真实场景中与 agent 的交互模式远比开发者预期的多样化，bug 暴露速度远超传统软件。
- **"评论区和私信里提的每一个需求，我都在看"指向了 agent 产品的关键渠道**：早期用户的反馈是 skill 质量和覆盖度迭代的最高效信号源。
- **Cron 预设（arXiv 日扫、引用追踪、截止提醒、组会准备、周报）是将 agent 从"工具"升级为"持续服务"的关键设计**：agent 不只是被动响应，而是主动在正确的时间做正确的事。

### 5. 产品数据公开是建立技术信任的有效策略

"431 个学术技能 · 28 个本地工具 · 13 个 API 工具 · 150 个 MCP 配置 · 6 个 Dashboard 面板 · 21 种消息卡片 · 1029 个单元测试"——这种精确的数字披露不是炫技，而是一种信任建立策略：

- 让技术用户（研究生/博士）能快速评估产品的工程深度。
- 暗含承诺：这些数字是可验证的，后续更新会继续公开对比。
- 对 Butler 的启示：对用户披露 skill/工具/测试的规模和质量状态，有助于建立长期信任关系。

## 可行动建议

1. **为核心 skill 建立最小测试基线**：参考科研龙虾"每个操作都有自动化测试守着"的思路，优先为飞书消息发送、文件操作、任务管理等高频 skill 编写基础回归测试，哪怕只是输入/输出格式校验。

2. **引入 skill 质量淘汰机制**：定期审查 skill 目录中的每个 skill 是否仍然有效、是否与当前架构兼容、是否在实际场景中被使用过，清理失效或冗余的 skill。

3. **在工作区关键操作路径上增加 Git 快照**：对 `BrainStorm/`、`工作区/` 等用户数据区域的写操作，考虑在每次 heartbeat 周期结束时自动做一次 git commit 快照，提供"后悔药"能力。

4. **设计 Butler 版的"Cron 预设"能力清单**：参考科研龙虾的 arXiv 日扫/引用追踪/周报等预设任务，为 Butler 梳理一套"用户开箱即用的周期性服务"模板，降低用户配置成本。
