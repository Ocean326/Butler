## 六、反爬诊断与抓取路径

### 6.1 XHS 反爬三层阻断（2026-03-18 三轮验证）

1. **SSR 层**：`__INITIAL_STATE__` 中 noteId 全部被置空（30/30，多次独立请求确认）
2. **访问层**：单帖 URL 需 noteId + **per-note** xsec_token；profile 级别 token 不可用于 explore 访问
3. **搜索引擎层**：XHS 帖子页 meta robots 限制，外部搜索引擎均无有效索引
4. **频率限制**：同一 IP 第二次请求 profile 页即降级为 7KB 登录壳页面
5. **站内搜索**：搜索结果页为纯 JS 渲染壳页面，WebFetch 无法获取数据

### 6.2 已验证的可行路径

**路径 A：xhslink 短链二步解析法**（单帖抓取的唯一可行路径）：

1. `requests.get(xhslink_url, allow_redirects=True)` → 302 Location 包含完整 `noteId + per-note xsec_token`
2. `requests.get(resolved_url)` → 200 OK，首屏数据完整
3. 交给 `social_capture.py` → 标准化 JSON/MD

**路径 B：WebFetch 主页短链**（本轮新发现，用于获取帖子列表）：

1. `WebFetch(https://xhslink.com/m/8o0d4Pck5Xw)` → 200 OK，返回主页 HTML
2. 可获取：所有可见帖子标题 + 点赞数 + 排列顺序（30 条，最新在前）
3. **不可获取**：noteId、xsec_token、帖子直链、发布日期、正文
4. 可用于：帖子列表核对、点赞数追踪、排序推断发布时间

### 6.3 历史排查记录（2026-03-18 破冰轮）

| 方案 | 结果 | 原因 |

### 6.4 瓶颈与突破

**仍存在的瓶颈**：获取单帖正文的前提是拿到 **xhslink 短链**，需要用户在小红书 App 中「分享 → 复制链接」。

**本轮突破**：WebFetch 主页短链可稳定获取帖子标题列表和点赞数，不依赖 noteId，适合作为监控 Simon 发帖动态的周期性检查手段。
