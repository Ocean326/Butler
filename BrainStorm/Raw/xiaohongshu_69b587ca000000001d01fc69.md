## Content

🦞我的科研龙虾一晃上线三天了，
为关注我的同学们汇报一下这 72 小时发生了什么～
	
Day 1 上线后第一件事：看群看到崩溃😡
有人说 Zotero 导入后标签数量对不上。有人说暗色主题下设置弹窗是白的，有人说文献库删了论文标签没跟着更新。
	
修。连夜修。三天睡了不到10个小时。
	
Day 2 补了 16 个新功能。
全是评论区和私信里大家要的：
→ 论文详情页可以展开了，不用再跳转
→ 任务支持子任务树了，可以拆解复杂的科研计划
→ 工作区加了文件预览，不用下载就能看
→ 科研雷达加了发现详情卡片，点进去能看到为什么推这篇给你
→ Cron 预设上线：arXiv 每日扫描、引用追踪、截止日期提醒、组会材料准备、周报——一键开启，不需要你配置任何东西
	
Day 3（今天）做了一件很重要但你们看不到的事。
测试用例从 318 个涨到了 1029 个。
	
为什么这很重要？
因为科研龙虾帮你管文献库、管任务、管文件。如果它自己不靠谱，帮你删错了论文、弄丢了笔记、搞混了引用…
	
1029 个测试，让文献库的每一个操作、任务系统的每一个状态变更、工作区的每一次文件写入，都有自动化测试守着。
你的数据，我用代码在你本地保证它安全。
	
还有一些你们可能关心的技术细节：
→ 重复论文检测上线了（DOI + 标题双重比对），不会再往文献库里加重复的
→ 所有工作区文件变更自动 Git 提交，误操作可以随时回滚
→ Windows Docker 部署打通了，不折腾 WSL2 也能用
现在的科研龙虾，跟三天前上线时已经不是同一个东西了。
	
431 个学术技能 · 28 个本地工具 · 13 个 API 工具 · 150 个 MCP 配置 · 6 个 Dashboard 面板 · 21 种消息卡片 · 1029 个单元测试
	
而且它还会继续进化。
你们在评论区和私信里提的每一个需求，我都在看。
	
最后说一句，评论区有人问我的skill 是不是 AI 生成的。
不是！431 个 skill，每一个都经过我人工遴选，而且上线三天我已经清理掉了 57 个质量不达标的。
