## 新信号

### 1. LangChain — The Anatomy of an Agent Harness

- **链接**: https://blog.langchain.com/the-anatomy-of-an-agent-harness/
- **要点**: 定义 Agent = Model + Harness 范式；系统推导 harness 核心组件（filesystem → bash/sandbox → memory/search → context rot 治理 → 长周期自主执行）；引用 Terminal Bench 2.0 数据证明仅改 harness 可从 Top 30 升至 Top 5。
- **与 Butler 关联度**: ⭐⭐⭐⭐⭐ (5/5) — Butler 本身就是 model 外层的 harness 系统，文章提出的 progressive disclosure、filesystem-as-memory、Ralph Loop 等模式与 Butler 的 planner-executor、skill 体系、上下文管理高度同构。
- **关联主线**: ① harness engineering ② context engineering ⑤ agent architecture

### 2. LangChain — Autonomous Context Compression

- **链接**: https://blog.langchain.com/autonomous-context-compression/
- **要点**: Deep Agents SDK/CLI 新增工具，让模型自主决定何时压缩上下文（而非依赖固定 token 阈值）；保留 10% 近期消息并摘要先前内容；全量历史持久化至虚拟文件系统。体现 "harness 应让位于模型自身能力" 的 bitter lesson 哲学。
- **与 Butler 关联度**: ⭐⭐⭐⭐⭐ (5/5) — Butler 的 heartbeat 长周期会话和 self_mind 续思场景直接面临 context rot 问题，自主压缩机制可直接借鉴。
- **关联主线**: ② context engineering ① harness engineering

### 3. LangChain — Introducing LangSmith Sandboxes

- **链接**: https://blog.langchain.com/introducing-langsmith-sandboxes-secure-code-execution-for-agents/
- **要点**: 基于 MicroVM 的安全沙箱，支持 agent 代码执行隔离；特性含池化自动缩放、Auth Proxy 避免 secrets 暴露、跨 sandbox 状态持久化、WebSocket 长连接流式输出；与 Deep Agents 框架原生集成。
- **与 Butler 关联度**: ⭐⭐⭐ (3/5) — Butler 当前不直接做代码执行，但未来若扩展到自动化脚本执行或 skill 沙箱运行，此方案提供安全边界参考。
- **关联主线**: ④ agent security / runtime

### 4. OpenAI — Harness Engineering: Leveraging Codex in an Agent-First World

- **链接**: https://openai.com/index/harness-engineering
- **发布日期**: 2026-02-11（非 3 天窗口内，但 BrainStorm 未收录过）
- **要点**: OpenAI Harness 团队用 Codex agent 生成全部代码（0 行手写），5 个月产出 ~100万行代码、~1500 PR；核心方法论包括 AGENTS.md 作为目录而非百科、progressive disclosure、机械化 linter 强制架构不变式、agent-to-agent code review、"垃圾回收"式持续重构。
- **与 Butler 关联度**: ⭐⭐⭐⭐⭐ (5/5) — 直接验证了 Butler 正在探索的"人类设计环境 + agent 执行"范式；AGENTS.md 设计哲学、progressive disclosure、repo-as-system-of-record 等模式可直接借鉴。
- **关联主线**: ① harness engineering ③ agent-first development ⑤ agent architecture
