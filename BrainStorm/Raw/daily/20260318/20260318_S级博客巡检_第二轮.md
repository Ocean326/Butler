## 新信号

### 1. DeepMind — Aletheia: 全自主数学研究 Agent

- **链接**: https://www.marktechpost.com/2026/03/13/google-deepmind-introduces-aletheia-the-ai-agent-moving-from-math-competitions-to-fully-autonomous-professional-research-discoveries/
- **论文**: https://github.com/google-deepmind/superhuman/blob/main/aletheia/Aletheia.pdf
- **发布日期**: 2026-03-13
- **要点**:
  - 基于 Gemini Deep Think 的三组件 agentic harness：Generator → Verifier → Reviser 迭代循环
  - 显式分离验证步骤（Verifier）使模型能发现生成阶段忽略的缺陷
  - 工具使用：Google Search + Web Browsing 防止引用幻觉
  - 成果：IMO-Proof Bench Advanced 95.1%（前纪录 65.7%）；自主解决 4 个 Erdős 猜想开放问题；Feng26 论文完全无人干预生成
  - 提出 AI 自主性分级（Level H → Level A × Level 0-4），类比自动驾驶分级
- **与 Butler 关联度**: ⭐⭐⭐⭐ (4/5) — Generator-Verifier-Reviser 模式与 Butler 的 planner-executor-verifier 架构高度同构；工具使用防幻觉策略、自主性分级对 Butler 治理设计有参考价值
- **关联主线**: ① harness engineering ⑤ agent architecture ④ agent eval/safety/autonomy

### 2. DeepMind — AutoHarness: 模型自生成约束，小模型胜大模型

- **链接**: https://agent-wars.com/news/2026-03-14-autoharness-llm-agent-code-constraints
- **论文**: arXiv:2603.03329
- **发布日期**: 2026-03-14
- **要点**:
  - Gemini-2.5-Flash 自动合成运行时约束代码（code harness），防止非法动作
  - 方法：Thompson sampling + tree search 在程序空间搜索，迭代优化约束代码
  - 两种变体：action-verifier（拒绝采样过滤非法动作）和 code-as-policy（直接生成决策策略，推理时零 LLM 调用）
  - 结果：action-verifier 在 145 个 TextArena 游戏中消除所有非法操作；code-as-policy 以更低成本超越 Gemini-2.5-Pro 和 GPT-5.2-High
  - 核心洞察：78% 的失败来自非法动作而非策略不好 → harness 的约束能力比模型本身更关键
- **与 Butler 关联度**: ⭐⭐⭐⭐⭐ (5/5) — 直接证明「harness > model」假说；自生成约束 = 自律系统的程序化实现；Thompson sampling 搜索约束空间的方法论可迁移到 Butler 的行为边界自动化校准
- **关联主线**: ① harness engineering ⑧ 自律系统与行为约束 ⑥ 自我进化/实验

### 3. HuggingFace — Holotron-12B: 高吞吐 Computer-Use Agent

- **链接**: https://huggingface.co/blog/Hcompany/holotron-12b
- **发布日期**: 2026-03-17
- **要点**:
  - H Company 基于 NVIDIA Nemotron-Nano-2 VL 后训练，专为 computer-use agent 优化
  - 混合 SSM（State-Space Model）+ Attention 架构，推理吞吐量比纯 Transformer 高 2x+
  - SSM 优势：线性复杂度、恒定状态空间（vs KV Cache 的线性增长），适合长上下文多图像 agentic 工作负载
  - WebVoyager Benchmark：从基座 35.1% 提升到 80.5%
  - 单 H100 上 100 并发时达 8.9k tokens/s
- **与 Butler 关联度**: ⭐⭐ (2/5) — Butler 当前不做 computer-use，但 SSM 架构对长上下文 agent 推理效率的启示有间接价值；关注 Nemotron 3 Omni 后续
- **关联主线**: ⑤ agent architecture（runtime/inference 视角）

### 4. NVIDIA — NeMo Agentic Retrieval Pipeline

- **链接**: https://huggingface.co/blog/nvidia/nemo-retriever-agentic-retrieval
- **发布日期**: 2026-03-13（HuggingFace 社区文章）
- **要点**:
  - ReACT 架构的迭代检索 pipeline：agent 循环 think → retrieve → refine
  - ViDoRe v3 #1（69.22 NDCG@10）、BRIGHT #2（50.90）
  - 关键工程决策：从 MCP Server 架构切换到进程内线程安全单例 retriever，消除序列化开销和部署复杂度
  - Agent 能缩小强/弱 embedding 模型之间的差距（gap shrinking 效应）
  - 成本权衡：平均 136s/query、760k input tokens/query → 后续方向是蒸馏到小模型
- **与 Butler 关联度**: ⭐⭐⭐ (3/5) — MCP → in-process singleton 的工程取舍对 Butler 的 tool/MCP 集成有参考；agentic retrieval 的 think-retrieve-refine 循环与 Butler 信息获取模式可对照
- **关联主线**: ① harness engineering ② context engineering
