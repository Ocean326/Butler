## 新信号

### 1. arXiv — AutoAgent: Evolving Cognition and Elastic Memory Orchestration

- **链接**: https://arxiv.org/abs/2603.09716
- **发布日期**: 2026-03-10
- **要点**:
  - 自进化多智能体框架，三核心组件：evolving cognition（结构化 prompt 级认知）、contextual decision-making（统一动作空间）、elastic memory orchestration（弹性记忆编排）
  - Elastic Memory Orchestrator 动态组织交互历史：保留原始记录 → 压缩冗余轨迹 → 构建可复用的情景抽象（episodic abstractions），降低 token 开销的同时保留决策关键证据
  - 闭环认知进化：将预期动作与观测结果对齐，持续更新认知、扩展可复用 skill，无需外部重训练
  - 在检索增强推理、工具增强 agent、具身任务等 benchmark 上优于静态和记忆增强基线
- **与 Butler 关联度**: ⭐⭐⭐⭐⭐ (5/5) — 弹性记忆编排（保留/压缩/抽象三层）与 Butler 的 heartbeat 上下文管理、self_mind 续思、BrainStorm Raw→Insight 沉淀管线高度同构；闭环认知进化 = Butler 正在探索的自我升级路径
- **关联主线**: ② context engineering ⑥ 自我进化/实验 ① harness engineering

### 2. Microsoft Research — PlugMem: 将原始交互转化为可复用知识

- **链接**: https://www.microsoft.com/en-us/research/blog/from-raw-interaction-to-reusable-knowledge-rethinking-memory-for-ai-agents/
- **论文**: arXiv:2603.03296
- **GitHub**: https://github.com/TIMAN-group/PlugMem
- **发布日期**: 2026-02 / 博客 2026-03-11
- **要点**:
  - 任务无关的即插即用记忆模块，三步流程：Structure（原始交互 → 命题知识 + 规范知识，组织为结构化记忆图）→ Retrieve（高层概念路由，检索任务对齐的知识单元而非长文本段落）→ Reason（蒸馏为简洁的决策就绪指导）
  - 核心洞察：更多记忆反而可能降低 agent 效能（信息过载悖论）；关键不是记多少，而是信息密度和决策相关性
  - 单一通用模块，在长期对话 QA、多跳知识检索、Web Agent 三个异质 benchmark 上不改设计即超越任务特定记忆方案
  - 以更少记忆 token 达到更好的决策效用
- **与 Butler 关联度**: ⭐⭐⭐⭐⭐ (5/5) — 直接对应 Butler 的 BrainStorm 管线（Raw 原始素材 → Insight 结构化知识 → 决策引导）；"信息密度 > 信息量"的洞察验证了 Butler 做 context rot 治理的方向；即插即用设计思路可参考用于 Butler skill 的记忆接口标准化
- **关联主线**: ② context engineering ① harness engineering ⑥ 自我进化/实验

### 3. arXiv — Multi-Agent Memory from a Computer Architecture Perspective

- **链接**: https://arxiv.org/abs/2603.10062
- **发布日期**: 2026-03-13（估计）
- **要点**:
  - 用计算机体系结构视角重构多智能体记忆：提出三层记忆层次（I/O → cache → memory），区分共享 vs 分布式范式
  - 识别"多智能体记忆一致性"为最紧迫挑战（类比 CPU 缓存一致性协议）
  - 行业趋势：分层三级记忆模型（全局/团队/个体）正在跨框架趋同；强调"范围化共享"优于全共享或全隔离
- **与 Butler 关联度**: ⭐⭐⭐⭐ (4/5) — Butler 的 planner-executor-subconscious 已是多层架构，三层记忆层次和一致性问题直接命中 Butler 在 heartbeat 多 branch 并行执行时的状态同步痛点；"范围化共享"与 Butler 的 memory policy（talk/heartbeat/self_mind 各层只读必要记忆）理念一致
- **关联主线**: ⑤ agent architecture ② context engineering
