## Content

很多团队做多 Agent，最先失控的往往不是模型，而是 skill 管理。
同一个 skill，既想给 Codex、Cursor、Claude Code 这类 Agent 应用使用，也想给 LangChain、LangGraph、AutoGen、CrewAI 这类框架里的多个 agent 复用。结果常常不是“复用”，而是“复制”——同一份 skill 被拷到多个项目里，一旦更新就要到处同步，最后越改越乱、越用越分叉。
MagicSkills 解决的正是这个问题。它不是再造一个 Agent 框架，而是给多 Agent 项目补上一层本地优先的 skill 基础设施。它的核心思路很简单：
一次构建 skill，供每个 Agent 复用。
具体来说，MagicSkills 会先把已安装的 skill 汇总到统一的共享 skill 池中；再为不同 Agent 创建各自的 Skills 集合，每个 Agent 只暴露真正需要的能力；最后再根据不同运行时的接入方式，同步到 AGENTS.md、CLAUDE.md，或者接成代码里的 tool / function。
这意味着什么？
意味着同一份底层 skill，不需要复制，就可以同时服务多个 Agent 应用和多个智能体框架；skill 的维护成本集中在一处，每次更新只需要维护共享池里的那一份；每个 Agent 也只看到自己真正需要的那组能力，边界更清晰，项目结构也更干净。
最有价值的一点是，新增一个运行时，不需要重写整套 skill，只需要从共享池里组合出新的 Skills 集合，再接上对应适配层即可。
如果你正在做多 Agent 系统，已经遇到 skill 复制、重复维护、能力分叉、运行时适配混乱这些问题，那 MagicSkills 很值得看。它解决的不是“模型更强”，而是一个更底层、也更容易被忽视的问题：
怎么让 skill 真正成为可复用资产，而不是一次次复制的目录。
