## 原始摘要：当 AI 开始自己做科研，人类的价值到底是什么（autoresearch 小红书笔记）

- **项目名称**：`autoresearch`
- **作者背景**：OpenAI 联合创始人、Tesla AI 前总监 Andrej Karpathy 发布的开源项目
- **发布时间线**：2025-03-07 发布，一周内 Star 数从 0 → 30,000+，当前已达 37,300+ Star（热度极高）
- **核心一句话**：给模型一个「竞技场」，让 AI 自己反复做实验、自己改代码，人类负责设定规则与边界。

### autoresearch 在做什么

- **功能本质**：一个让 AI 自动跑实验、自动调参、自动改代码的极简框架。
- **使用方式**：
  - 人类只需要给一个高层目标，例如「让模型更聪明」。
  - 系统每约 5 分钟自动发起一次实验，根据结果自动保留好方案、丢弃差方案。
  - 形成一个持续的「进化」过程，第二天人只需要看结果报告和曲线。
- **代码规模**：核心逻辑约 630 行代码，强调「极简可读、可 fork 可改」。

### 笔记中给出的实测结果（作者自己的实验）

- **初始基线**：指标值 1.506。
- **调参 100 轮后**：1.266（约 16% 提升）。
- **加长训练至 90 分钟**：1.021（约 19% 提升）。
- **发现隐藏参数 MATRIX_LR=0.02 后**：1.010。
- **2 小时长训练**：0.992（首次突破 1.000）。
- **3 小时长训练**：0.975（当前最佳）。
- **总提升幅度**：从 1.506 → 0.975，整体约 35%，作者判断还没碰到架构性能上限（估计约 0.95）。

### 作者从 autoresearch 得到的核心启发

- **AI 复现人类多年成果的速度**：
  - 在约 17 小时的自动实验中，AI 重新「发现」了人类在 8 年内逐步发展出的若干技术（如 RMSNorm 等）。
  - 这并不是「凭空创造」，而是说明在给定规则和搜索空间下，AI 可以极快地遍历组合，找到和人类类似的好的设计。
- **人类角色的迁移**：
  - 人类不再是「亲自做每一次实验的人」，而是「设计实验竞技场的人」。
  - 真正的价值在于：定义规则、约束、奖励信号、搜索空间，而不是亲手跑每一次训练。
- **跨领域迁移**：
  - 作者明确指出：这套思维不只适用于模型训练。
  - 在产品、营销、增长等场景，一样可以通过「搭建自动试验场 → 让 AI 自己在规则内做 A/B 测试与策略搜索」来完成。

### 笔记中提到的本地配置（可作为现实可行性的参考）

- **硬件配置**：RTX 5060 Ti，显存约 16GB（显存占用仅 6.1GB）。
- **系统环境**：Windows + PowerShell + `uv`。
- **使用者背景**：AI 产品经理转型中，零 ML 理论基础，说明上手门槛相对友好。

### 笔记给出的快速上手步骤（原文指令式）

1. `git clone https://github.com/karpathy/autoresearch`
2. 安装 `uv`，运行 `prepare.py` 下载数据。
3. 将配置中的 `DEPTH` 调整为 `8` 以适配普通显卡。
4. 执行 `uv run train.py`，让系统自动跑实验。
5. 第二天查看 `results.tsv`，基于结果继续迭代配置与搜索空间。
