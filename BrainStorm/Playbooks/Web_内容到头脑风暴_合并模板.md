## 一、单篇 Web 内容 → 结构化头脑风暴模板

（以下内容来自原 `Web_内容到头脑风暴_模板.md`）

## 这个模板用来做什么

> 场景：当你在知乎专栏 / 技术博客 / 小红书 / 公众号等位置看到一篇「值得沉淀」的文章，希望：
> - 把它从「一篇网页」变成「一份可复用的结构化头脑风暴」；
> - 方便 Butler / IDE 助手在后续心跳或对话中快速引用这份结构，而不是重新读长文。
>
> 用法：为每一条重要 Web 内容单独复制一份本模板，填完后：
> - Raw 原文 / OCR 等仍放在 `BrainStorm/Raw`（或工作区抓取目录）作为事实真源；
> - 本文件则作为「结构化头脑风暴视图」，可在 BrainStorm 中索引，也可被任务 / 对话直接引用。

---

## 一、基础信息区（必填）

- **标题**：`原文标题 / 自定义工作标题`
- **来源平台**：`知乎专栏 / 博客 / 小红书 / 公众号 / 其他`
- **作者 / 账号**：`例：某某 / 某团队`
- **发布时间**：`YYYY-MM-DD（若未知可写「未标明」）`
- **原始链接**：`https://...`
- **备份链接 / 截图说明**：`可选：飞书文档 / 图床 / 本地截图目录`
- **当前获取方式**：`手工整理 / web-note-capture-cn / web-image-ocr-cn / 其它工具`
- **对应 Raw / OCR 文件**：`BrainStorm/Raw/... 或 工作区/... 抓取文件路径`
- **当前处理状态**：`草稿 / 已初步结构化 / 已进入 MEMORY / 已转成协作手册`

> 备注：这里是「元数据 + 真源入口」。但后续头脑风暴与行动线索只改本文件，不回写 Raw。
