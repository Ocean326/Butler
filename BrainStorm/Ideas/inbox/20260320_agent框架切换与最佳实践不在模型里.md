# Agent 框架切换与最佳实践不在模型里

## 闪念

很多 Agent 开发者“悄悄换框架”，可能不是因为新框架更酷，而是因为它更像一个已经把脑内经验固化进去的 Agent 壳。

## 为什么值得记

- 这条内容直接把一个很关键的分界线说透了：
  - AI 程序员可以帮你写应用
  - 但它还不等于会帮你搭一个可靠的 Agent 系统
- 它提示了一个更深的问题：
  - Agent 的核心 best practice 可能主要不在模型里，而在少数开发者的运行时经验、调试套路、上下文治理和信任设计里
