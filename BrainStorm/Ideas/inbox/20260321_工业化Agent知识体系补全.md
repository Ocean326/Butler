# 工业化 Agent 知识体系补全

## 闪念

Butler 现在已经有不少 Agent 工程直觉，但还缺一层更系统的“工业化开发知识底座”。

问题不只是“怎么把一个 Agent 做出来”，而是：

- 怎么把它做成生产级系统。
- 怎么把它做成可扩展、可观测、可恢复、可治理的系统。
- 怎么在多智能体、长链路、复杂工具调用、高并发和长周期演化下，仍然不失控。

我现在看到的一个很典型触发器，是各种“大厂面经”里反复出现的那些问题：

- 生产级 Agent 架构怎么设计。
- 规划-执行-反思闭环怎么做。
- 分级记忆和缓存怎么做。
- 向量库怎么选。
- Function Calling 的可靠性怎么保证。
- RAG 的 chunk / recall / rerank 怎么设计。
- 多工具串行延迟怎么压。
- 高并发弹性伸缩怎么做。
- 监控、告警、Trace 怎么打通。

这些表面上像面试题，实际上背后是一整套工业化 Agent 的知识树。

## 为什么值得记

- Butler 当前更强的是“架构嗅觉”和“工程直觉”，还不够强的是“系统化工业知识总图”。
- 这套知识如果不补，后面很多设计只能靠局部经验，容易在关键阶段反复重学。
- 它也不是单条主线能吃下的内容，而是横跨：Harness、Runtime、Memory、RAG、Tooling、Observability、Infra、Governance 的跨主线体系。

## 这条线真正要研究的，不只是 12 道题

我现在更想研究的是一整棵“工业化 Agent 知识树”：

### 1. Runtime / Harness

- 生产级 Agent runtime 的标准分层是什么
- agent loop / turn / thread / checkpoint / durable execution 的稳定抽象是什么
- 单 Agent、multi-agent、workflow、supervisor、subagent 的边界怎么划

### 2. Context / Memory

- working memory / episodic memory / semantic memory / task state 如何分层
- recent window、summary、external memory、cache、retrieval 之间怎么配合
- memory 的压缩、失真、污染、去重、回放问题怎么治理

### 3. Tool Use / Function Calling

- 工具描述、schema、参数边界、异常分类、重试策略怎么设计
- tool-call fail loop、错误工具重复调用、参数幻觉怎么治理
- 什么时候该让模型“自由调用”，什么时候该强约束和显式 router

### 4. RAG / Retrieval / Vector DB

- chunk、index、hybrid search、rerank、metadata filter、contextual retrieval 如何选型
- Milvus / Pinecone / Chroma / Qdrant / pgvector 等的真实工业边界是什么
- 多租户、冷热分层、成本、延迟、召回质量怎么权衡

### 5. Latency / Throughput / Cost

- 多工具串行延迟怎么拆
- 哪些步骤该并行，哪些步骤该 cache，哪些步骤该降级
- token、工具调用、检索、网络、队列、推理预算怎么一起优化

### 6. Reliability / Durable Execution

- 长任务中断后如何恢复
- checkpoint、idempotency、side effects、replay、resume 怎么做
- rainbow deployment、灰度切流、版本并存如何接入 agent runtime

### 7. Observability / Eval / Telemetry

- 单次请求从用户输入到最终产出，完整链路怎么 trace
- span、event、tool call、state transition、token/cost/latency/error 怎么建模
- 离线 eval、在线监控、自动告警、失败案例库如何形成闭环

### 8. Infra / Scale

- Agent 服务在高并发下怎么做弹性伸缩
- K8s HPA / KEDA / queue-based scaling / async worker 什么时候用
- GPU 推理层、应用层、工具执行层、检索层如何解耦扩缩容

### 9. Governance / Safety / Entropy

- 什么时候需要 human-in-the-loop
- 权限门控、审批、预算控制、沙箱、自升级边界怎么设计
- agent 系统长期运行的熵增如何治理

### 10. Productization / Team Engineering

- 从 demo 到 production 的迁移路径是什么
- 哪些能力必须先做：trace、receipt、failure taxonomy、replay、smoke baseline
- 团队如何共享 prompt / skill / tool / eval / case library

## 当前想到的研究方法

不只做“资料收藏”，而是按四层证据组织：

1. **官方工程博客 / 官方文档**
   先抓 Anthropic、OpenAI、LangChain/LangGraph、OpenTelemetry、Kubernetes、Temporal、Pinecone、Milvus、Chroma 等一手材料。

2. **GitHub 仓库与 README / Discussions**
   看真实仓库暴露出的抽象、部署方式、限制与社区常见问题。

3. **论坛 / 讨论区 / 工程博客**
   补一手文档里没有写透的“踩坑层”经验。

4. **回写 Butler 设计映射**
   每次外部知识都要回答：这对 Butler 当前架构意味着什么。
