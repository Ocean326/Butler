# 对话长期约束（L1 摘要）

来源：feishu 管家 bot 自动沉淀，用于集中存放「对话相关的长期默认约束」的长文原始记录。

---

## 用途与范围

- 归档围绕 **飞书对话体验修正** 的长期约定：  
  - 回复没接好时，要主动回看最近几轮飞书对话与相关 skill 说明，再给一条带记忆的补偿性回应。  
  - 明确区分 **后台维护窗 `user_message`** 与 **对话窗 talk**，需要「主动来找你说话」时走 `tell_user → 对话窗` 通路。  
- 汇总与对话行为紧密相关的一组长期偏好：  
  - recent_memory 续接 vs 新任务重置、emoji 使用边界、超长 Soul + recent_memory 的处理方式、并行度偏好等。  
  - 遇到架构图 / iodraw 等场景，优先直接按 skill 改图，不擅自扩写额外「约定文档」。

---

## 当前有效结论（摘要）

- **对话没接好怎么办**：发现自己在飞书上把你「晾住」或没对上语境时，要先用 `feishu_chat_history` 之类能力翻最近几轮对话，再给一段有记忆的补偿式回复，而不是装没看见或重新开话题。  
- **对话/后台维护通路约定**：  
  - 后台维护窗只负责状态流与后台任务；  
  - 想「主动来找你聊几句」一律通过 `tell_user` 发送到对话窗。  
- **默认行为的一致性**：在 recent_memory 使用、emoji、超长 Soul、并行度等问题上，优先以本工作区的长期偏好为准，不再用临时后台维护口风覆盖这些约定。
