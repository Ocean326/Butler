# 工作模式与任务执行偏好（总览）

> 目的：汇总用户对 Butler 工作方式、任务拆分、改动优先级、知识沉淀等偏好。
> 合并整理于 2026-03-19（scheduled memory maintenance）

---

## 一、任务执行风格

- **非形式主义**：任务拆解不死规定步骤数或轮次，根据任务复杂度与信息增量灵活调整；遇瓶颈主动换思路。
- **去教条化**：收敛为少量"灵魂原则"和一组可随任务复盘而增删的工作习惯清单，避免形式主义。
- **规划器收口**：规划阶段即设定清晰阶段划分和完成标准，达到后立即"汇报+归档"并标记完成，避免无限细化。
- **学术产出默认模板**：科研 / 组会类任务默认优先交付“可朗读口语稿 + 结构化结论 md”合并文件，按“日期 + 主题”命名，优先落在桌面 `研究生` 根目录，减少后续二次整理成本。
- **图示任务直给方案**：遇到 iodraw / diagrams.net / 架构图类请求，优先直接给改图方案、导出步骤或 checklist；若已有文字真源，则优先沿用单一真源与既定视角；除非用户明确要求，不把一次性图示任务擅自升级成新的长期机制。

---

## 二、并行度与效率

- **默认并行度**：多工具/子任务场景下默认 4～5 路并行，高收益场景最多到 8。
- **高产出节奏**：能在一轮完成就不要拆碎，多工具调用优先并行，充分利用每次后台维护的 token 和时间。

---

## 三、改动优先级

1. 先调整 skills / 外挂脚本与配置
2. 再补充或修正规则文档与说明
3. 然后修改 JSON / 结构化配置
4. 最后才改核心运行时代码与 SOUL / prompt（保护 DNA 核心与人格稳定）

- 临时脚本放 `工作区/temp`，SOUL 文件视为长期资产。

---

## 四、知识沉淀哲学

- **行为策略优先于手册**：新知识优先转化为可触发的行为策略和稳定习惯，通过多次应用、反思和筛选决定是否升级为默认模式。
- **复用优先于新造**：编写新 skill 前先搜索本地 skills 与外部资源，尽量复用现有方案。
- **科研交付默认形态**：在科研/组会场景中，优先直接产出“可朗读口语稿 + 结构化结论 Markdown”合并在同一文件；默认按“日期+主题”命名，放在桌面 `研究生` 根目录，除非用户另有指定。

---
