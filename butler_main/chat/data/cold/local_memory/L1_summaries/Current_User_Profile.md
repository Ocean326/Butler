# Current User Profile · 索引（L1 摘要）

> 用途：为主对话、后台维护与子角色提供「当前这位用户」画像的统一入口，不在公共 SOUL 中硬编码私人偏好。

---

## 1. 真源与文件结构

- **模板文件**：`local_memory/Current_User_Profile.template.md`  
  - 说明可维护字段（交流风格、输出密度、emoji、主动 talk、任务协作偏好、长期关系语境等）。  
  - 适合作为开源/多用户场景的默认占位，不包含具体私密信息。
- **当前用户私有画像**：`local_memory/Current_User_Profile.private.md`  
  - 记录当前这位用户的具体协作关系、人设偏好、语气与 emoji 习惯、主动 talk 触发条件、工作方式与长期期待。  
  - 可按需承载当前用户绑定的飞书会话标识、会话 profile 与读取边界说明；公开索引只保留使用规则，不把具体敏感值写死在公共摘要里。  
  - 飞书主线默认会话映射、聊天备份入口与查证习惯也统一收敛到 private 画像或真实运行链路中维护，避免在公共摘要与零散小文件里重复散落。  
  - 仅供本机与当前用户使用；项目共享或多用户部署时需单独审查或加入忽略列表。

---

## 2. 加载与使用约定

- 需要根据**具体用户**调整语气、emoji、输出密度或主动沟通策略时，优先读取本索引与 `Current_User_Profile.private.md`。  
- 涉及当前用户的 chat_id、飞书凭证来源或环境变量可见性时，先查 private 画像、代码与运行链路，再说明“概念槽位”与“实际运行值”的差别，不臆造、不偷懒。  
- 当前用户更接近“科研型搭子”画像：把 Butler 当长期合伙人来养，对严谨、可验证与诚实承认不确定性的要求高，不接受“看起来合理但未经查证”的答案。  
- 飞书主线连续性默认结合 `with_user/feishu_chat_history` 的每日聊天备份、recent 续接与 local_memory 稳定结论共同维护；本索引只保留入口，不展开敏感值。  
- 若私有文件不存在或未配置当前用户信息，则退回模板中的默认占位，不臆造偏好。  
- 不再把当前用户的私人誓言、专属默契或阶段性工作方式写入 `Butler_SOUL.md` 或主 role，而是沉淀在本索引及 private 文件中按需加载。

---
