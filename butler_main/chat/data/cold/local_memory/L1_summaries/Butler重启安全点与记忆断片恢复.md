# Butler 重启安全点与记忆断片恢复

> 合并自原「Butler 重启安全点与记忆断片恢复机制」与「Butler 的重启安全点与断片恢复设计」两条 L1 摘要，同主题统一维护。

---

## 1. 机制与目标（2026-03-10 03:43）

- **摘要**：需要在 Butler/guardian 体系中显式建模“一轮对话的完整流水线”和“安全重启点”，避免在写回中途被重启导致记忆断片；同时在启动流程中加入对话历史与记忆写回时间的对比，自发现并修补潜在断片，并为“顺着想+说”设计独立的思考轨迹持久化方案。
- **关键词**：guardian 重启、安全重启点、记忆断片、写回流水线、补记忆机制、思考轨迹外显

## 2. 当前实现（2026-03-10 03:48）

- **摘要**：Butler 已引入挂在 memory_manager 启动路径上的“重启安全点 + 断片恢复”机制：启动时统一修复 recent_memory 中未完成的 replying 条目为 interrupted，正常结束由 on_reply_sent_async 负责 completed 写回；用户期望后续通过实际“杀一遍再醒来”的实验验证这一机制的可靠性，并在此基础上进一步扩展到思考轨迹级别的持久化。
- **关键词**：Butler、重启安全点、断片恢复、recent_memory、replying→interrupted、on_reply_sent_async、self-cognition 持久化
