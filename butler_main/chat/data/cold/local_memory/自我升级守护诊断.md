### 1.2 当前失败现象（来自 restart_guardian_latest.json 与 recent_memory）

- **触发来源**：watchdog-healthcheck-main-down  
- **触发原因**：对话主进程未运行（PID=46308 不存活）  
- **level2-manager-restart / level3-pacemaker-stop-all / level3-pacemaker-start**：均 `exit_code=1`，`output` 为空  
- **detected_issues**：权限不足导致启动或写文件失败；Python 运行时异常（Traceback）  
- **建议位置**：butler_bot.py:550（实为 `sys.exit(main())` 出口，非业务逻辑）

### 1.3 根因归纳

1. **Traceback（err 日志）**  
   - 最新主进程 err（butler_bot_20260308_001.err.log）中 Traceback 为 **2026-03-08 05:02:39** 的 **KeyboardInterrupt**（用户/终端中断），属正常退出路径。  
   - 后续 multiprocessing spawn 的 KeyboardInterrupt 为父进程退出后的连带现象。  
   - **结论**：当前无「反复崩溃型」Python 异常；报告中的「Python 运行时异常」多为历史 Traceback 的归纳，或 forensics 从 err 尾部解析出的旧内容。

2. **权限 / 执行策略**  
   - 守护调用 `manager.ps1 restart/stop/start` 时若 PowerShell 执行策略为 Restricted，或当前用户对 run/logs/研究管理工作区 无写权限，会导致步骤失败且可能无清晰 stderr（output 为空）。  
   - err 中曾出现 `manager.ps1:130 ParserError: MissingEndCurlyBrace`：多为 **PowerShell 输出编码/中文乱码** 导致 stderr 混入中文后解析显示异常，不一定表示 ps1 语法错误（manager.ps1 第 130 行为正常 `}`）。

3. **早间「未找到 manager.ps1」**（scripts/butler_bot 内 recent_memory 06:21～06:26）  
   - 报告路径为 `...\scripts\butler_bot\scripts\butler_bot\manager.ps1`，说明当时传入守护的 workspace 可能为「项目根/scripts/butler_bot」且某处仍按 scripts/butler_bot 再拼了一级，或规范化只做一次、未循环到真正项目根。  
   - 当前最新 report 中 workspace 已为项目根，该问题在当日后续时段未再出现。

## 3. 可执行修复步骤（供用户或聊天主进程执行）

### 3.1 权限与执行策略（优先）

1. **管理员 PowerShell** 中执行：  
   `Get-ExecutionPolicy -List`  
   若 CurrentUser 或 LocalMachine 为 Restricted，可放宽：  
   `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

2. **验证目录写权限**（项目根 = 研究生 目录）：  
   - `scripts\butler_bot\run`、`scripts\butler_bot\logs`  
   - `研究管理工作区`、`研究管理工作区\governance\self_upgrade_reports`  
   在 PowerShell 中：  
   `Test-Path "项目根\scripts\butler_bot\run"; New-Item "项目根\scripts\butler_bot\run\test_write.tmp" -Force; Remove-Item "项目根\scripts\butler_bot\run\test_write.tmp" -ErrorAction SilentlyContinue`

### 3.2 手动复现 level2 步骤

在**项目根**（研究生）下执行：  
`powershell -NoProfile -ExecutionPolicy Bypass -File "scripts\butler_bot\manager.ps1" status`  
再执行：  
`powershell -NoProfile -ExecutionPolicy Bypass -File "scripts\butler_bot\manager.ps1" restart butler_bot`  
观察是否有 stderr 输出、是否 exit 0；若失败，将完整错误信息用于进一步排查。

### 3.3 确认主进程状态与 err 日志

- 主进程状态文件：`scripts\butler_bot\run\butler_bot_main_state.json`（pid、updated_at）。  
- 最新 err：`scripts\butler_bot\logs\butler_bot_20260308_001.err.log`（或当前日期段）。  
若主进程当前未运行，Traceback 为历史；**先完成 3.1、3.2 后再启动主进程**，避免重启链再次因权限/执行策略失败。

### 3.4 若需代码级修复（由聊天主进程批准后执行）

- **memory_manager 传入守护的 workspace 规范化**：在 `_launch_restart_guardian` 内对 `workspace` 做与 restart_guardian_agent 一致的「若为 .../scripts/butler_bot 则规范为项目根」，避免少数启动场景下传入错误路径。  
- **restart_guardian_agent 循环规范化**：在 `_normalize_workspace_to_project_root` 中循环直到路径末尾不再是 `scripts/butler_bot`，避免双重 scripts/butler_bot 导致 manager.ps1 路径错误。  
以上需写入 **研究管理工作区/background_maintenance_upgrade_request.json**，由用户批准后由聊天主进程执行（不在此后台维护内直接改 scripts/butler_bot）。
