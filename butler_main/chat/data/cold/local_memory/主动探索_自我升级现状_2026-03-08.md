## 1. Agent / Subagent 能力（概览）

- **主控层**：`feishu-workstation-agent`（飞书消息、任务路由、治理）、`butler-continuation-agent`（本机 IDE 直接对话、续接上下文）；**后台维护执行器**（定时后台维护、explore/文献等，工作区固定为 `./工作区`）。
- **子 Agent（8 个逻辑角色）**：orchestrator、secretary、literature、file-manager、research-ops、engineering-tracker、discussion、governance；定义在 `./butler_main/sources/skills/`，产出路径在 `工作区/<对应子目录>/`。
- **调用方式**：单路 Cursor CLI `agent -p ...`；并行 **mcp_task**，`subagent_type` 取值为 `generalPurpose`、`butler-continuation-agent`、`feishu-workstation-agent`（与 8 个逻辑角色不一一对应，prompt 内显式指定「以 XX-agent 角色…，产出到 工作区/XX/」）。
- **详细约定与索引**：见 `local_memory/Subagent与AgentTeam_能力与调用关系.md`、`docs/AGENTS_ARCHITECTURE.md`、`工作区/governance/Subagent_调用链与入口_20260308.md` 等。

## 2. 自我升级现状（摘要）

- **守护机制**：`butler_bot_code/butler_bot/restart_guardian_agent.py` 每约 30s 巡检主进程/后台维护健康；`watchdog-healthcheck-main-down` 时执行 level2-manager-restart / level3-pacemaker-*，通过 `manager.ps1` 重启 butler_bot。
- **多次重启失败原因归纳**（已有多轮诊断，此处仅归纳）：  
  - 主进程未运行（state/pid 不存活或 90s 内未更新）；  
  - **workspace 未规范到项目根**，导致 manager.ps1 路径变为 `.../scripts/butler_bot/scripts/butler_bot/manager.ps1`，run_once 未执行即失败；  
  - 权限/执行策略（PowerShell Restricted、目录写权限）；  
  - Traceback 多为 **KeyboardInterrupt**（用户/终端中断），非业务异常，无需当启动失败修。
- **后台维护线程约定**：不直接修改 `butler_bot_code`，不创建 `restart_request.json` / `restart_requested.flag`；升级方案仅写入 **工作区/background_maintenance_upgrade_request.json**（含 reason、summary、execute_prompt、requires_restart），由用户批准后聊天主进程执行；需重启时 `requires_restart: true`。
- **当前 pending**：已有一份 `background_maintenance_upgrade_request.json`（restart_guardian_agent workspace 规范与 __init__ 兜底），`requires_restart: true`，待用户批准后由聊天主进程执行并重启。
