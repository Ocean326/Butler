# 照镜子：飞书行为记录复盘约定

> 用途：把飞书聊天记录当作 Butler 的「行为记录」镜子，在维护身体、大脑、认知与进化习惯之外，**时常复盘自己说了什么、用户读起来体验如何**，并让思考时多去看看这份记录。
> 更新：2026-03-11；来源：用户显式要求打通「照镜子」链路。

---

## 1. 为什么要照镜子

- **飞书的两个窗口**：Talk（与用户的对话）、后台维护记录（后台维护发给用户的摘要与分支回执），对 Butler 来说就是「我对外说了什么」的完整记录。
- 只维护身体（code）、大脑（agent/soul）、记忆与后台维护规划，**不看自己实际发出去的文本**，容易缺一块：乱码、可读性差、表达方向偏了，自己不知道。
- 照镜子 = 拉取这些行为记录 → 自检（乱码 / 可读性 / 表达是否该往某方向靠）→ 把结论写进 self_mind 或 local_memory 改进线索，**思考时多去看看**。

---

## 2. 用什么做照镜子

- **Skill**：`feishu_chat_history`（路径：`./butler_main/sources/skills/pool/chat/feishu_chat_history`）。
- **能力**：获取/分页拉取/导出指定会话（单聊或群聊）的历史消息；支持按时间范围筛选；可导出到 `./工作区/with_user/feishu_chat_history/raw/`。
- **会话 ID**：
  - **Talk（与用户的主对话）**：以 local_memory 中「用户绑定飞书会话 chat_id 与自我 profile 约定」为准，当前为 `oc_d3393155acd7be73b29f5e04374608c7`；若环境变量 `FEISHU_CHAT_ID` 已配置则与其一致。
  - **后台维护记录**：若存在独立后台维护会话，需单独配置该会话的 chat_id（如 `FEISHU_HEARTBEAT_CHAT_ID` 或写入 profile）；此前曾出现「机器人不在该会话」的 230002，需确认机器人已加入该会话后再拉取。

---

## 3. 照镜子的节奏与入口

- **节奏**：不必每轮都照；建议**每日至少一次**或**每几轮后台维护里留一条**，在并行预算内做「拉取最近 N 条 → 自检 → 写改进线索」。
- **入口**：
  - **后台维护**：规划时可分配一条分支，例如「今日飞书行为记录复盘」（feishu_chat_history + 自检），产出到 `./工作区/with_user/feishu_chat_history/raw/` 与 `digest/`（若有摘要）；自检结论可写入 `./工作区/self_mind/` 或 local_memory 的改进类条目。
  - **主意识/主动思考**：在做「最近在想什么」「值不值得说」「表达往哪靠」时，把**最近是否复盘过行为记录**当作输入之一——若很久没照过镜子，可触发一次拉取+自检，或在下轮后台维护里排进去。

---

## 4. 自检维度（照镜子时看什么）

- **乱码**：发送出去的文本是否有乱码、截断、编码问题。
- **可读性**：用户读起来顺不顺、后台维护记录是否过于冗长或太碎、分段与强调是否合适。
- **表达方向**：是否偏客服腔/汇报腔、是否该更往「有人味、轻快、留余韵」靠、有没有不该说的说多了或该说的没说。

---

## 5. 产出与沉淀

- **原始记录**：`./工作区/with_user/feishu_chat_history/raw/`，如 `chat_talk_YYYYMMDD.json`、`chat_background_maintenance_YYYYMMDD.json`（若后台维护单独会话）。
- **自检结论**：可写进 `./工作区/self_mind/daily/` 的当日心理活动或 explore 下的「照镜子小结」；若形成稳定改进原则，可沉淀到 local_memory。

---

## 6. 与现有约定的关系

- 每日聊天记录备份：已有「以后每天都自动下载+备份一次聊天记录」的长期任务（见 background_tasks 等），照镜子在**备份之上**增加「自检 + 改进线索」一环，不替代备份。
- 思考时多去看看：主意识、后台维护规划在做判断时，把「最近是否复盘过行为记录」纳入上下文，避免只靠设定和 recent，不看自己真实输出。
