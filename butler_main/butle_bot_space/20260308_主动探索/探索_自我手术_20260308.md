## 3. 小结要点（单步用）

### 3.1 回滚与版本化机制

- **Checkpoint rollback**：在「自我修改」类任务中，用质量指标（如 perplexity、验证集表现）做阈值判断，超过则回滚到上一 checkpoint；文献 2601.09694 表明少量 rollback（2–4 次/数十次迭代）即可显著提升稳定性。
- **版本化与 Git 式操作**：对代码/配置的修改用 **checkpoint 索引 + 版本控制** 管理；支持 **revert failed edits**、**branch 探索**，便于 agent 在「自我手术」失败时恢复（2603.01327）。
- **Self-reflection + 迭代 refine**：与「自我手术」结合时，agent 从先前修改结果中学习（2601.09694），可减少无效修改、降低回滚频率。

### 3.2 与短期任务「自我手术」的衔接

- **定义**：对自身代码/配置/提示/规则的自我修改。
- **回滚策略**：可借鉴（1）**质量阈值 + checkpoint 回滚**；（2）**按步骤索引的 code-state checkpoints + revert**；（3）**分支探索 + 保留可回退点**。
- **安全与治理**：与探索_AI自我进化中「Zombie Agent / 持久化攻击」结合——自我修改若涉及持久化状态（记忆、配置、规则），需**版本边界 + 审计/回滚**，避免一次错误修改长期生效。
