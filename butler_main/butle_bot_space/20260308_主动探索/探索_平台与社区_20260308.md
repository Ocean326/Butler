## 1. Agent 层平台与协议（可复用入口）

| 名称 | 类型 | 简介 | 链接 |
|------|------|------|------|
| **MCP** | 协议/标准 | Model Context Protocol，连接 AI 应用与外部系统（数据源、工具、工作流）的开放标准；Cursor、Claude、ChatGPT、VS Code 等均支持。 | https://modelcontextprotocol.io/ |
| **LangGraph** | 框架 | 以图结构构建 resilient 语言 Agent（LangChain 生态）。 | https://github.com/langchain-ai/langgraph |
| **CrewAI** | 框架 | 角色扮演、自主 AI Agent 编排，强调多 Agent 协作。 | https://github.com/joaomdmoura/crewAI |
| **AutoGen** | 框架 | 微软出品，面向 Agentic AI 的编程框架。 | https://github.com/microsoft/autogen |
| **Dify** | 平台 | 开源 Agentic Workflow 开发平台；可视化编排、工具与数据连接、自托管部署；与 LangGraph/CrewAI 等代码式编排形成互补。 | https://github.com/langgenius/dify \| https://docs.dify.ai/ |

- **与本工作区**：当前已用 Cursor + MCP（子 Agent 通过 mcp_task 等调用）；orchestrator 分派与各 specialist agent 对应「多 Agent 编排」；可与上述框架做对比参考，不要求替换现有架构。**Dify** 于 2026-03-08 补充并入本表，详见 `探索_Dify平台_20260308.md`。

## 2. 文献入口（arXiv，仅 Agent 层）

- **检索式**：`multi agent orchestration LLM`（all fields）
- **近期代表性工作**（2026 年 2–3 月，侧重编排/框架/通信，不涉及模型训练）：
  - **OrchMAS** (2603.03005)：两层级多模型编排，动态构建领域感知推理管道、专家 Agent 与提示，支持异构 LLM、动态重规划。
  - **MASPOB** (2603.02630)：多 Agent 系统提示优化（bandit + GNN），样本高效、考虑拓扑耦合。
  - **ESAA** (2602.23193)：Event Sourcing for Autonomous Agents，意图与状态分离、确定性编排、AGENT_CONTRACT、多 Agent 并发案例。
  - **Cross-Agent Semantic Flows** (2603.04469)：多 Agent 执行感知攻击检测，跨 Agent 语义流重建。
- **用途**：后续细化「编排模式」「子 Agent 边界」「自我手术回滚」时可从中抽取设计模式与术语。
