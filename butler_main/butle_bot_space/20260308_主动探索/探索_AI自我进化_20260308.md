## 2. 小结要点

### 2.1 概念与机制

- **自我进化的常见形态**：  
  - **Self-play / co-evolution**：Generator–Solver–Verifier 三角色（如 2603.02218）；或 Generator–Solver 双角色 co-evolution（如 Tool-R0），从零数据实现工具调用能力。  
  - **可持续进化的条件**：合成数据需带来**可学习信息增益**，否则 self-play 易 plateau；设计上需考虑非对称 co-evolution、容量增长、主动引入外部任务/上下文（2603.02218）。  
  - **提示与权重的分工**：E-SPL（2602.14697）联合进化 **system prompt**（变异、交叉、选择）与 **RL 权重更新**，声明性知识放 prompt、程序性知识放权重，在推理与 agent 任务上带来一致收益。

- **记忆与长期状态**：  
  - **Self-evolving memory**：UMEM 等将「记忆抽取」与「记忆管理」联合优化，避免只优化管理而把实例噪声当稳健记忆。  
  - **安全与回滚**：自我进化 agent 会跨会话更新内部状态（写/用长期记忆），带来持久化攻击面（Zombie Agent：一次间接注入即可长期控制）；防御需超越「单会话 prompt 过滤」，考虑记忆演化与回滚策略（2602.15654）。

### 2.2 与短期任务「自我进化 / 自我手术」的衔接

- **自我进化**：对应文献中的 self-play、co-evolution、prompt evolution、memory evolution；可细化「可学习信息增益」「prompt–权重分工」「跨会话状态与安全」。  
- **自我手术**：对应对自身代码/配置/提示/规则的自我修改；文献中与「回滚」衔接的是持久化攻击与防御（记忆/状态的可控更新与还原），可下次在「自我手术」方向单独做检索（回滚策略、版本化、审计）。
