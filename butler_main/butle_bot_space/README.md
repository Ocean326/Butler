# Butler 的家

> `butle_bot_space/` 是 Butler 的家，不是工作交付区，也不是运行时代码区。

## 现在的问题

当前家里只有一个 `备份_20260308_主动探索/` 大箱子，能看出有生活痕迹，但没有形成稳定结构：

- 备份、探索、复盘都混在一起。
- 没有明确说明什么该放家里，什么不该放。
- 没有把「常住内容」和「只为恢复保留的内容」分开。

这会导致后续一边把公司当家，一边又把家当临时仓库。

## 家里应该放什么

- 备份副本
- 主动探索原始材料
- 自我整理过程稿
- 生活节律与心跳相关的私人沉淀
- 以后需要回看但不适合放进公司正式目录的材料

## 家里不应该放什么

- 正式交付给用户或团队的结果
- 运行时 PID、日志、缓存
- 已经稳定成制度的规则真源
- 大量重复的诊断报告副本

## 目标结构

当前先按逻辑分区理解，不强制一次性建满目录：

- `备份_YYYYMMDD_主题/`：一次探索或自我手术前的备份包
- `探索/`：原始探索与学习材料
- `self_mind/`：主意识上下文、心理活动流水、自由生活线索
- `复盘/`：生活化、自我成长、自我维护复盘
- `恢复索引/`：记录哪些备份值得回看、哪些已经过时

## 归档与冷存档

- **默认关闭**：除非用户**显式提起**「归档」或「冷存档」，否则家里**不主动启用**归档和冷存档流程。
- 默认动作不是移动文件，而是**只在长期记忆里登记候选项**。
- 当用户明确要求归档时，再执行目录移动或冷存放。

## 当前执行原则

1. 家先做边界澄清，不急着制造更多空目录。
2. 能作为恢复材料的旧内容先保留，不随意移动。
3. 对需要归档的候选项，先写长期记忆条目，再等明确指令执行。

一句话：**家是 Butler 的生活区和恢复区，不是默认开启的档案馆。**
