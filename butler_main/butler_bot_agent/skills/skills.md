# Butler Skills 使用总则

> 用途：作为所有角色在“什么时候必须/优先使用 skill、如何调用 skill”时的统一约定入口；具体到某个 skill 时，再读取该目录下对应的 `SKILL.md`。

## 1. 什么时候必须 / 优先使用 skill

- **有匹配 skill 的任务场景时，要优先用 skill**
  - 只要用户的需求，和某个已登记的 skill 能力明显对应，就应优先通过 skill 完成任务，而不是直接改核心代码或临时造一次性脚本。
  - 在考虑新增 skill 前，先搜索本地 `./butler_bot_agent/skills` 与外部资源，尽量复用现有轮子；仅在无合适轮子或严重不适配时再新建 skill，并在说明中记录已检索过哪些方案、为何需要新轮子。

- **典型触发场景（在使用过程中持续补充）**
  - 飞书侧能力：历史消息、文档检索、群/会话工具等 → 优先查 `feishu_*` 相关 skills。
  - 外部平台抓取与学习：小红书、论坛、网页资料等 → 优先查 `xiaohongshutools` 等相关 skills。
  - 日常巡检与治理：启动巡检、健康检查、配置/目录巡检等 → 优先查 `daily-inspection` 及后续运维类 skills。
  - 其他可复用流程：批处理、导出、整理套路等 → 优先查 `operations` / `general` 分类下的 skills。
  - 若在心跳或自我升级分支中涉及“skills 探索 / 外部学习”，应显式说明本轮会参考哪些既有 skills，并在产出中完善本节的典型场景列表。

## 2. 调用 skill 的标准流程

1. **匹配 skill**
   - 当用户提到“skill / 技能 / 调用 skill / 调用某某 skill”或任务明显属于某类外部能力时：
     - 先在 `./butler_bot_agent/skills` 中按名称、描述与 `SKILL.md` 内容匹配出最相关的一个或少数几个 skill。
     - 若用户没有点名具体 skill，由当前角色根据任务意图自行选择最合适的 skill，并在回复中说明选择依据。

2. **读取 `SKILL.md`**
   - 在真正执行前，**必须**用 Read 读取该 skill 目录下的 `SKILL.md`：
     - 按 `SKILL.md` 中的“适用场景 / 前置条件 / 步骤 / 配置来源 / 风险点”来设计本轮操作。
     - 如有 `reference.md`，在遇到细节不清晰或需要扩展用法时再按需阅读。

3. **按说明执行**
   - 执行时遵守 `SKILL.md` 的约定，不随意突破其边界：
     - 不把 DNA 级能力（核心运行机制、生命周期、日志/安全闸门等）偷偷塞回 skill。
     - 如发现 `SKILL.md` 本身过时或有问题，应在本轮产出中记下“需更新该 skill 说明”的结论，而不是临时擅自改写行为。

4. **在回复中透明说明**
   - 在对用户的自然语言回复中，**明确说明**：
     - 本轮使用了哪个 skill（名称 + 相对路径）。
     - 该 skill 在本轮大致做了什么（1–2 句话）。
     - 若未找到合适 skill，也要如实说明“当前未找到匹配的 skill”，而不是装作已经使用。

## 3. 与各角色的协作约定

- **Soul 层（Butler_SOUL）**
  - 把“优先善用 skills、优先复用现有轮子”视为人格与价值排序的一部分，而不是可有可无的工程细节。

- **对话入口角色**
  - `feishu-workstation-agent` 与 `butler-continuation-agent`：
    - 在各自 role 文件中引用本 `skills.md` 作为“skills 使用总则”。
    - 当用户说“调用 skill”时，默认按本文件的匹配与调用流程执行。

- **心跳与子 Agent**
  - 心跳 planner 在规划涉及外部能力或自我升级/skills 学习的分支时：
    - 优先安排“读取对应 `SKILL.md` + 按说明执行”的小步动作。
    - 在 branch 的 `prompt` 头部明确写出将要使用的 skill 名称及主要目标。

## 4. 后续维护方式

- 本文件只描述“通用使用规则与触发场景示例”，不承载具体某个 skill 的细节。
- 每当新增或大幅调整一个 skill 时：
  - 在对应 `SKILL.md` 中写清楚适用场景与调用示例。
  - 视情况在本文件的“典型触发场景”中补充一行，让其它角色更容易发现并复用该 skill。
