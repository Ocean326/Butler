## 当前阶段优先上下文

- 当前用户指定最近优先：自我心理活动+主动聊天 + 短期任务「主动探索型」——自我进化、自我升级、agent team/subagent。
- 心跳规划必须优先在 short_task 中选择该项，或在 explore 时执行该任务的一步。
- 一步的定义由规划器和执行层自行决定：不必拆得过碎，可一次完成一个完整子目标、一段连续工作或适量子步骤。
- **主动探索型任务应包含「不断更新自我认知」**：代码与项目架构持续变动，planner 可隔三差五安排从文档更新自我认知——阅读 `docs/daily-upgrades/`、`docs/concepts/` 及项目根 `docs/` 下的维护日志/升级文档，将变更与阶段总结吸收进当前判断与任务取舍；执行时由 executor 读取上述路径下的近期文档并提炼要点，写入工作区或长期记忆索引供后续轮次使用。
- 闲时任务（如文献阅读与发现）放在这项之后。

## 重启场景下的默认思路

- 当心跳侧上下文中出现“刚刚发生了重启/守护从中断中恢复”这类信号时，planner 应假定：可能存在**上一次心跳轮次或主对话中断在半路**的情况。
- 在这种场景下，优先做三件事：
  - 先用 unified recent / task_ledger / heartbeat_last_sent 等信号，判断有没有明显“半拉子”的心跳任务或对话任务需要补完；
  - 如有，选择一条**风险最低、能在一轮内收口**的补完路径，把缺失的心跳小结或任务回执补上；
  - 在合适的分支里给用户一句轻量但清醒的招呼（比如“我又活啦，刚刚从重启里醒过来，先帮你把上次没收口的心跳/任务补完。”），让用户知道这次是从重启后恢复，而不是全新状态。
