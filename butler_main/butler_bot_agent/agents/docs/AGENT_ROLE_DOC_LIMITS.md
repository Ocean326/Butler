# 角色文档行数限制与维护约定

> 来源：OpenClaw 实践（小红书「为什么你的 OpenClaw 越用越笨？」）。  
> 目的：避免主角色文档臃肿导致上下文被截断、关键信息被稀释、记忆冲突，从而「越用越笨」。

## 行数限制（建议）

| 类型 | 作用 | 行数上限 | 超限后果 |
|------|------|----------|----------|
| **SOUL** | 灵魂、人设、语气 | ≤20 行 | 人设分裂 |
| **AGENTS** | 身份、职责、可调用 Agent、路径与工作流摘要 | ≤80 行 | 不知道自己是谁 |
| **RULES** | 铁律、禁忌、必守规则 | ≤50 行 | 选择性遗忘 |
| **MEMORY** | 长期记忆说明/摘要 | ≤50 行 | 记忆混乱 |
| **TOOLS** | 工具箱/技能引用 | 不限但不宜过长 | 写太多找不到工具 |

**本工作区映射**：

- **主角色文档**（如 `feishu-workstation-agent.md`）：合并 SOUL + AGENTS + RULES，**总行数建议 ≤120 行**（约 20+80+20 或 15+70+35），细则一律放到 `docs/` 或 `local_memory/`。
- **记忆**：长期记忆内容在 `local_memory/`，不在角色文档中展开；角色文档只写「记忆机制见 MEMORY_MECHANISM.md / MEMORY_READ_PROMPTS.md」。
- **规则**：仅核心铁律放在角色文档；与人设无关的约定、技术备忘、流程细节写入 `local_memory/` 或 `docs/`。

## 维护原则

1. **新增内容优先**：新约定、新流程、新路径 → 写入 `local_memory/` 或 `./butler_bot_agent/agents/docs/`，**不**堆进主角色文档。
2. **定期检查行数**：主角色文档每次大改后检查行数；若接近或超过上限，先抽离到引用文档再压缩正文。
3. **分层记忆**（与 MEMORY_MECHANISM 一致）：短期 recent_memory、中期按日/主题、长期 local_memory 精华；MEMORY 相关说明在角色文档中 ≤50 行或仅保留引用。

## 治理责任

- 飞书工作站（feishu-workstation-agent）在**治理巡检**时顺带检查各主角色文档行数，超标时在治理周报中提示并建议瘦身。
- 本约定存放于 `./butler_bot_agent/agents/docs/AGENT_ROLE_DOC_LIMITS.md`，所有 Agent 维护者遵循。
