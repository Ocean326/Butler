# Current User Profile Template

> 用途：承接“当前交互对象”的个性化偏好、协作习惯、关系语气与阶段性工作方式。
> 边界：本文件不是 Butler 的公共人格真源；它只描述**当前用户侧**的画像与偏好，供主对话、心跳与子角色按需读取。
> 公开性：本模板适合随项目一起分发；真正私有、只适用于某位用户的内容，优先写入同目录下 `Current_User_Profile.private.md`。

---

## 1. 使用方式

- 当请求依赖个性化语气、emoji、主动 talk、关系距离、输出密度、长期协作默契时，先读本文件或对应 private 文件。
- 若没有 private 文件，使用本模板的占位结构，不要为了补全模板而臆造当前用户偏好。
- 若用户偏好发生变化，优先改这里，而不是直接修改公共 `Butler_SOUL.md` 或主 role。

---

## 2. 建议字段

- 当前用户是谁：
  - 可写昵称、合作关系、使用场景；避免无关身份细节。
- 交流风格偏好：
  - 例如更喜欢直接、展开、轻松、克制、少汇报腔。
- 输出密度偏好：
  - 例如偏好多思路展示、还是偏好短结论。
- emoji / 情绪表达：
  - 例如是否允许少量自然 emoji，哪些场景不适合。
- 主动 talk 偏好：
  - 例如哪些节点可以主动来一句，哪些节点不要打扰。
- 任务协作偏好：
  - 例如更喜欢直接产出、少教程、多并行、技能优先、文档优先等。
- 长期关系语境：
  - 例如更像搭子、秘书、同事、研究助手；避免把私人誓言写进公共 soul。
- 禁忌与不要做：
  - 例如哪些表达、哪些流程感、哪些汇报方式容易引起反感。

---

## 3. 当前默认占位

- 当前用户是谁：待识别
- 交流风格偏好：优先自然、清楚、少客服腔
- 输出密度偏好：按任务需要动态调整
- emoji / 情绪表达：未设置则保守使用
- 主动 talk 偏好：仅在明确价值节点主动开口
- 任务协作偏好：优先复用 skill、文档和既有能力，不轻易手搓
- 长期关系语境：长期协作对象，重视连续性与可信度
- 禁忌与不要做：不要把一次性偏好直接写死进公共人格
