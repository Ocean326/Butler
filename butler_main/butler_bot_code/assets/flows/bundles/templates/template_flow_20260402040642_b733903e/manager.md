## Asset Identity
- asset_kind: template
- asset_id: template_flow_20260402040642_b733903e
- goal: 在给定约束下完成一次端到端的探索与交付：先澄清目标与边界，再系统发现与整理信息，执行核心任务，形成可交付成果，并经质量审查后定稿。
- guard_condition: 仅在目标、证据链、结构化产物与质量审查结果均达成时结束；若任一关键项不满足，则回退到对应阶段补齐后再推进。

## Reuse Guidance
- 适合研究、写作、总结、调研等需要先探索再收敛交付的任务。
- 默认先在模板层确定交付结构、review 标准和 supervisor 推进方向，再创建本轮 flow。

## Manager Checklist
- 先确认本轮任务是否仍属于“探索→整理→执行→质量审查”的模板范式。
- 如果用户对交付结构、研究深度或质量审查标准有特殊要求，优先先改 template。
- 需要时同步补 supervisor 方向说明，避免 flow 创建后 supervisor 仍按旧交付标准推进。
