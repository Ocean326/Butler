# 飞书机器人接收命令接入指南

> 目标：在飞书中 @机器人 或私聊发送命令，由管家bot（Cursor CLI）处理后回复

## 一、与当前 Webhook 的区别

| 能力 | 自定义机器人 Webhook（当前） | 企业自建应用（需新建） |
|------|------------------------------|------------------------|
| 接收消息 | ❌ 不支持 | ✅ 支持 |
| 发送消息 | ✅ 仅 Webhook 推送 | ✅ API 回复 |
| 配置复杂度 | 低 | 中 |
| 本地运行 | ✅ | ✅ 长连接无需公网 |

**结论**：要接收飞书命令，必须创建**企业自建应用**，不能使用自定义 Webhook 机器人。

---

## 二、创建企业自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 创建企业自建应用：**创建应用** → **创建企业自建应用**
3. 填写应用名称（如 `管家bot`）
4. 进入应用 → **凭证与基础信息** → 复制 **App ID** 和 **App Secret**

### 开启机器人能力

1. **功能** → **机器人** → 启用
2. 配置：勾选「接收群聊中 @机器人 消息」「接收单聊消息」（按需）
3. 保存

### 事件订阅（长连接）

1. **事件与回调** → **事件配置**
2. **订阅方式**：选择 **使用长连接接收事件**
3. 点击 **添加事件** → 搜索 `接收消息` → 选择 **接收消息 v2.0**（`im.message.receive_v1`）
4. **先启动本地服务**（见下节），长连接在线后再点击保存

### 发布与使用

1. **版本管理与发布** → 创建版本 → 申请发布（企业内可用）
2. 将机器人添加到目标群：群设置 → 群机器人 → 添加 → 从应用目录选择你的应用

## 三、本地服务运行

### 1. 安装依赖

```bash
pip install lark-oapi -U
```

### 2. 配置

复制 `butler_bot_code/configs/butler_bot.json.example` 为 `butler_bot_code/configs/butler_bot.json`，填写 app_id、app_secret：

```python
{
	"app_id": "cli_xxxx",
	"app_secret": "xxxx"
}
```

### 3. 启动

**推荐（统一管理）：**
```powershell
cd "c:\Users\Lenovo\Desktop\研究生\Butler\butler_main\butler_bot_code"
.\manager.ps1 start butler_bot
```

**或直接运行：**
```bash
cd "c:\Users\Lenovo\Desktop\研究生\Butler\butler_main\butler_bot_code"
python .\butler_bot\butler_bot.py --config .\configs\butler_bot.json
```

控制台出现 `connected to wss://...` 表示长连接成功。

### 4. 回到飞书后台保存事件配置

长连接在线后，在应用 **事件与回调** → **事件配置** 中保存。

### 5. 进程管理

使用统一管理脚本（推荐）：
```powershell
cd "c:\Users\Lenovo\Desktop\研究生\Butler\butler_main\butler_bot_code"
.\manager.ps1 list              # 列出已注册机器人
.\manager.ps1 status            # 查看运行状态
.\manager.ps1 start butler_bot   # 启动（后台）
.\manager.ps1 stop butler_bot    # 停止
.\manager.ps1 stop --all        # 停止全部飞书机器人
```

详见 `butler_bot_code/README.md`。

## 四、使用方式

- 在群聊中 @机器人 + 你的问题/命令
- 或与机器人私聊发送
- 机器人会调用管家bot（Cursor CLI）处理，并将结果回复

**注意**：Agent 执行可能需要数分钟。若 15 秒内模型无输出，会回复「正在输入…」；完成后再发一条完整回复。
