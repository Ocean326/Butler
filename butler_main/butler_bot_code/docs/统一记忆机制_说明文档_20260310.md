# 统一记忆机制说明文档

更新时间：2026-03-10

## 这次改了什么

这次不是只加了一层 recent 摘要，而是把 Butler 的记忆链路重排成了四层：

1. 对话与心理活动先进入统一 short-term schema。
2. 【潜意识】会把一轮里的心理活动、关系/情绪信号、任务信号拆成 companion entries。
3. 需要长期保留的内容会被提升为 structured long-term memory。
4. heartbeat 规划时不再只看任务文件和 heartbeat observation，而是读统一 Soul + role + unified recent + local index。
5. planner 和 executor 的结果会统一交给潜意识整理，再决定写回 beat recent 和 long-term memory。

## 现在 heartbeat 真正会读什么

heartbeat planner 现在会读：

1. Butler 的 Soul 摘录
2. 当前 role 摘录
3. 对话侧 unified recent
4. 心跳侧 recent execution flow
5. local memory 的当前结论、历史演化、适用情景

这意味着 heartbeat 已经可以从长期记忆里“恢复一个明确事项”，而不是只会围着 heartbeat_tasks.md 打转。

## local memory 现在长什么样

新写入的长期记忆会带这三个核心槽位：

1. 当前结论
2. 历史演化
3. 适用情景

同时在 `L0_index.json` 里保留结构化字段，供 heartbeat planner 和后续查询直接读取。

## 潜意识现在怎么 work

潜意识这轮已经不只是“把 entry 规范化”了，而是会：

1. 给一轮对话生成 talk 主 entry。
2. 生成 mental / relationship / task 三类 companion entries。
3. 在用户出现长期偏好、默认约定、关系偏好变化、内部反复整理等信号时，隐式补出 long-term candidate。
4. 给出 trigger level，区分普通 recent 写回和需要 long-term 提升的轮次。
5. 在 heartbeat 中接收 planner 判断和 executor 结果，把它们整理成可继续恢复的“阶段结论”。

## 主 agents 和 sub-agents 现在怎么分

现在的分层更接近“一个人”的内部结构：

1. `feishu-workstation-agent` / `butler-continuation-agent`：对外说话和接住场景。
2. `heartbeat-planner-agent`：心跳里的思维和规划。
3. `subconscious-agent`：潜意识，负责把经历整理成可再调用的内在记忆。
4. `sub-agents/*`：具体场景执行者，比如 literature、secretary、heartbeat-executor。

## 你现在能直接感受到的变化

1. 后续 heartbeat 规划更容易续上之前的长期事项。
2. 关系偏好、风格约定、工作方式这类内容更容易被真正记住。
3. local memory 不再是一堆难以恢复的散摘要，而是更接近“当前有效结论 + 来源演化”。

## 仍然没做完的部分

1. 历史演化目前还是启发式生成，不是完整版本链。
2. role excerpt 现在是 heartbeat 视角下对 `feishu-workstation-agent` 的复用，还可以继续细化成独立 planner role。
3. heartbeat 执行结果回灌长期记忆还可以继续做得更深。

## 建议你接下来怎么用

1. 如果你希望某类偏好长期生效，可以继续自然表达，不需要再死记规则词。
2. 如果你要观察 heartbeat 是否真的在恢复长期事项，可以看 planner 产出的 `reason` 和 `user_message` 是否明确提到“从长期记忆恢复”。
3. 如果你发现某条长期记忆写得不够像“当前结论”，下一轮可以继续推动我把 profile 生成做得更像版本式知识卡。
