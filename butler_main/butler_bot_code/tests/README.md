# Butler Tests

`tests/` 只保留可重复运行的回归测试。

## 目录约定

- `test_*.py`
  - 标准自动化回归测试
  - 每次重构后必须修复、更新或删除其中已经失效的用例
- `manual/`
  - 手工脚本、自测脚本、一次性排查脚本
  - 不参与默认 `pytest` 收集
- `manual/obsolete/`
  - 已过时但暂时保留参考价值的历史测试
  - 对应能力若已下线、私有 API 已删除、或运行链已改写，应优先移到这里而不是继续污染默认套件

## 维护原则

1. 新增主机制、目录重构、协议变更时，要同步补一条对应回归测试。
2. 过时测试不要留在默认收集区假装可用；要么修到可跑，要么移入 `manual/`，要么删除。
3. 优先维护稳定的单元/协议测试，避免把强依赖外部环境的脚本混进默认测试集。
4. 至少覆盖这些高频回归面：
   - 主链路导入是否正常
   - 目录/路径真源是否仍一致
   - chat / orchestrator 的运行状态与入口是否仍可用
   - 微信 recent memory 的会话隔离是否仍可用
   - 关键文本协议与回复裁剪逻辑是否未回退

## 运行方式

```powershell
.venv\Scripts\python.exe -m pytest
```
