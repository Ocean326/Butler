# research-iteration

Use this recipe when a campaign already has partial findings and needs another bounded cycle toward a stronger answer.

## Recommended role order

1. `planner`
2. `literature-scout` for targeted gap fill
3. `evidence-synthesizer`
4. `report-writer`
5. `evaluator`

## Parallel window

- `report-writer` should wait until `evidence-synthesizer` has produced a usable matrix.
- Additional `literature-scout` instances may run in parallel if they own different search slices.

## Final verdict

- `evaluator`

## Required evidence

- updated iteration plan from `planner`
- gap-focused source shortlist from `literature-scout`
- updated claim-evidence matrix from `evidence-synthesizer`
- bounded draft from `report-writer`
- verdict and next-step guidance from `evaluator`

## Avoid

- unbounded multi-week drift
- writing beyond the evidence base
- treating "more research" as the default verdict
