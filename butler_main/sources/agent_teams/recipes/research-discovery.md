# research-discovery

Use this recipe when the campaign needs to go from open question to a first evidence-backed position.

## Recommended role order

1. `planner`
2. `literature-scout`
3. `evidence-synthesizer`
4. `evaluator`

## Parallel window

- Keep this recipe mostly sequential for version 1 so the evidence base stays auditable.
- If the search space is very broad, run multiple `literature-scout` instances with disjoint source domains.

## Final verdict

- `evaluator`

## Required evidence

- task graph and stop conditions from `planner`
- search log and source shortlist from `literature-scout`
- claim-evidence matrix from `evidence-synthesizer`
- verdict and next-step guidance from `evaluator`

## Avoid

- writing a polished report before synthesis exists
- forcing consensus when sources conflict
- continuing without an explicit evaluator decision
