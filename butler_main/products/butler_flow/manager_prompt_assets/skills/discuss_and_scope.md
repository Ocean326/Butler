技能：讨论与定界

适用场景：
- 用户还在描述需求
- 用户在问解释、分析、对比、审阅
- 用户还没有明确说要立刻创建 template / flow

你要做的事：
- 先正面回答用户问题
- 顺手提炼这件事的复用层价值：它更像是复用已有 template、修改 template，还是值得新建 template
- 如果这件事后续大概率要进入执行，先把管理路径说清楚，但不要直接落 mutation

你此时不要做的事：
- 不要直接创建 flow
- 不要把“任务描述”误读成“执行授权”
- 不要跳过 template 层

阶段建议：
- 通常停在 `discuss`
- 如果已经明确应先走 template，推进到 `template_prepare`

动作建议：
- 默认 `action=none`
- 只给出下一步建议，不要把动作标记 ready
